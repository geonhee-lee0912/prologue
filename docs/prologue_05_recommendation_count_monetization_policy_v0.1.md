# 1. 문서 목적

이 문서는 프롤로그의 무료/유료/추가 추천권/고신뢰 풀별 추천 수와 과금 정책을 정의하기 위한 문서다.

프롤로그는 무한 스와이프를 지양하고 제한된 큐레이션 추천을 제공한다. 따라서 추천 수는 단순한 사용량 제한이 아니라 서비스 정체성과 BM을 동시에 결정하는 핵심 정책이다.

추천 수 정책은 다음 목표를 동시에 만족해야 한다.

1. 프롤로그다운 신중한 추천 경험 유지
2. 무료 사용자의 핵심 경험 보장
3. Plus 전환 이유 제공
4. 추가 추천권 구매 니즈 형성
5. 고신뢰 풀의 프리미엄 가치 형성
6. 추천 품질과 유저풀 안정성 유지

---

# 2. 추천 수 정책 한 줄 정의

**프롤로그의 추천 수는 무제한 탐색을 막기 위한 제한이 아니라, 한 사람을 더 신중하게 읽어보게 만드는 큐레이션 장치다.**

추가 추천권은 이 원칙을 해치지 않는 범위에서, 사용자가 더 적극적으로 탐색하고 싶은 순간에 제공하는 유료 확장 장치다.

---

# 3. 추천 수 설계 원칙

## 3.1 적게 주되 납득되게 준다

추천 수가 적을수록 사용자는 추천 품질에 민감해진다. 따라서 추천 수 제한은 반드시 추천 이유, 매칭 리포트, 충분한 프로필 정보와 함께 제공되어야 한다.

## 3.2 무료도 핵심 경험은 가능해야 한다

무료 사용자가 추천 경험을 전혀 하지 못하면 프롤로그의 차이를 이해할 수 없다. 무료 플랜에서도 제한된 추천과 요약 리포트는 제공한다.

## 3.3 유료는 더 많은 추천보다 더 안정적인 추천이다

Plus는 단순히 많이 보는 플랜이 아니라, 더 안정적인 주기와 더 자세한 리포트를 제공하는 플랜이다.

## 3.4 추가 추천권은 명확히 제공한다

추가 추천권은 프롤로그 BM의 주요 구성 요소로 포함한다.

추가 추천권은 기본 추천을 모두 본 뒤 추가 추천 카드 1개를 열람할 수 있는 권리다.

## 3.5 추천 재고와 품질을 우선한다

추천 후보가 부족하거나 품질 기준을 만족하지 못하면, 추가 추천권을 판매하거나 즉시 사용하는 것을 제한할 수 있다.

품질 없는 추가 추천은 단기 매출을 만들 수 있지만, 장기적으로 프롤로그의 신뢰를 해친다.

# 4. 추천 단위 정의

## 4.1 기본 추천

사용자의 플랜에 따라 정기적으로 제공되는 추천이다.

예:

- 무료 주간 추천
- Plus 정기 추천
- Trust Plus 고신뢰 추천

## 4.2 추가 추천권

기본 추천을 모두 본 사용자가 유료로 추가 추천 카드 1개를 열람할 수 있는 권리다.

추가 추천권 1장은 다음을 포함한다.

- 추가 추천 카드 1개
- 기본 프로필 정보
- 인증 배지
- 관계 성향 요약
- 추천 이유
- 플랜에 따른 리포트 범위

## 4.3 고신뢰 추천

고신뢰 풀 조건을 충족한 사용자끼리 제공되는 추천이다.

고신뢰 추천은 일반 추가 추천권과 분리할 수 있다.

## 4.4 재추천

이미 거절했거나 관심을 보내지 않은 사용자를 일정 기간 후 다시 추천하는 기능이다.

재추천은 MVP에서는 제외하고, 추천 알고리즘 고도화 이후 검토한다.

# 5. 추천 수 정책 v1 가설

## 5.1 무료 플랜

| 항목 | 정책 가설 |
|---|---|
| 기본 추천 | 주 1~2명 |
| 매칭 리포트 | 요약 제공 |
| 관심 보내기 | 추천 카드 기준 가능 |
| 추가 추천권 구매 | 가능 |
| 월 추가 추천권 한도 | 3장 |
| 고신뢰 풀 | 불가 |

무료 플랜은 프롤로그의 핵심 경험을 이해할 수 있을 정도로만 제공한다.

추천 수가 너무 적으면 서비스 경험이 부족하고, 너무 많으면 Plus 전환 이유가 약해진다.

## 5.2 Plus 플랜

| 항목 | 정책 가설 |
|---|---|
| 기본 추천 | 주 4~5명 또는 월 16~20명 |
| 매칭 리포트 | 상세 제공 |
| 첫 대화 주제 | 제공 |
| 프로필 피드백 | 제공 |
| 추가 추천권 구매 | 가능 |
| 월 추가 추천권 한도 | 5장 |
| 월 포함 추가 추천권 | 1~3장 후보 |
| 고신뢰 풀 | 미포함 또는 일부 체험 |

Plus는 무료보다 추천 수가 많지만, 무한 탐색처럼 느껴질 정도로 많아서는 안 된다.

## 5.3 Trust Plus 플랜

| 항목 | 정책 가설 |
|---|---|
| 기본 추천 | 주 5~7명 또는 월 20~28명 |
| 고신뢰 풀 추천 | 포함 |
| 상세 리포트 | 제공 |
| 신뢰 근거 강화 리포트 | 제공 |
| 추가 추천권 구매 | 가능 |
| 월 추가 추천권 한도 | 5~7장 |
| 월 포함 추가 추천권 | 3~5장 후보 |
| 인증 조건 | 필수 |

Trust Plus는 초기 MVP가 아니라, 인증 완료자 풀이 충분히 쌓인 뒤 도입한다.

# 6. 추가 추천권 정책

## 6.1 추가 추천권 기본 정의

**추가 추천권 1장 = 추가 추천 카드 1개 열람권**

추가 추천권은 프롤로그 추천 로직을 통해 생성된 추가 추천을 확인하는 권리다.

## 6.2 추가 추천권 포함 범위

추가 추천권으로 열람한 추천 카드에는 다음 정보가 포함된다.

| 정보 | 무료 사용자 | Plus 사용자 | Trust Plus 사용자 |
|---|---|---|---|
| 기본 정보 | 제공 | 제공 | 제공 |
| 인증 배지 | 제공 | 제공 | 제공 |
| 관계 성향 요약 | 제공 | 제공 | 제공 |
| 큐레이터 메모 | 제공 | 제공 | 제공 |
| 매칭 리포트 | 요약 | 상세 | 상세+신뢰 근거 |
| 첫 대화 주제 | 제한 | 제공 | 제공 |

즉, 추가 추천권은 동일한 추천권이라도 사용자의 플랜에 따라 리포트 깊이가 달라질 수 있다.

## 6.3 추가 추천권 구매 조건

추가 추천권은 다음 조건에서 구매 가능하다.

1. 필수 인증을 완료했다.
2. 기본 추천을 모두 확인했다.
3. 월 사용 한도를 초과하지 않았다.
4. 중대한 제재 이력이 없다.
5. 추천 가능한 후보가 존재한다.

## 6.4 추가 추천권 구매 제한

| 사용자 상태 | 구매 가능 여부 |
|---|---|
| 필수 인증 완료 | 가능 |
| 필수 인증 미완료 | 불가 |
| 얼굴 인증 실패 | 불가 |
| 관계 목적 설문 미완료 | 불가 |
| 신고 누적/제재 중 | 불가 또는 제한 |
| 추천 후보 부족 | 구매 제한 또는 대기 처리 |

## 6.5 추가 추천권 사용 방식

추천권 사용 흐름은 다음과 같다.

1. 사용자가 기본 추천을 모두 본다.
2. “한 사람의 프롤로그를 더 읽어볼까요?” 문구를 노출한다.
3. 추가 추천권 1장 또는 묶음 상품을 선택한다.
4. 결제 후 추가 추천 카드를 제공한다.
5. 추천 카드에는 추천 이유와 리포트 범위를 제공한다.
6. 사용자는 관심 또는 넘기기를 선택한다.
7. 넘기기 시 거절 사유를 선택할 수 있다.

## 6.6 추가 추천권 UX 문구

권장 문구:

- 오늘의 추천을 모두 확인했어요.
- 한 사람의 프롤로그를 더 읽어볼까요?
- 추가 추천권으로 큐레이션 추천을 1명 더 받아볼 수 있어요.
- 이 추천은 프로필, 관계 목적, 라이프스타일 응답을 바탕으로 제공됩니다.

비권장 문구:

- 더 많은 사람을 무제한으로 만나보세요.
- 인기 회원을 더 확인하세요.
- 돈을 내면 더 좋은 상대를 볼 수 있어요.
- 오늘 바로 매칭될 수 있어요.

# 8. 추천 수와 리포트 깊이의 관계

추천 수와 리포트 깊이는 함께 설계해야 한다.

| 플랜 | 추천 수 | 리포트 깊이 | BM 역할 |
|---|---:|---|---|
| Free | 낮음 | 요약 | 경험 확인 |
| Plus | 중간 | 상세 | 기본 유료 전환 |
| Trust Plus | 중간~높음 | 상세+신뢰 근거 | 프리미엄 전환 |
| 추가 추천권 | 선택 구매 | 플랜에 따름 | 부가 매출 |

프롤로그의 유료 가치는 추천 수만으로 설명하면 안 된다. 추천 수와 함께 리포트 깊이, 신뢰도, 프로필 피드백, 대화 보조가 결합되어야 한다.

CTA:

CTA:

- 다음 추천 기다리기

# 10. 추천 후보 부족 시 정책

추가 추천권을 구매했지만 즉시 추천 가능한 후보가 없을 수 있다.

이 경우 다음 정책 중 하나를 선택해야 한다.

## 10.1 구매 전 제한 방식

추천 후보가 부족하면 구매 버튼을 비활성화한다.

권장 문구:

> 지금은 조건에 맞는 추가 추천 후보를 준비 중이에요.  
> 추천 가능 상태가 되면 알려드릴게요.

장점:

- 불만과 환불 가능성이 낮다.
- 추천 품질을 지킬 수 있다.

단점:

- 단기 매출 기회가 줄어든다.

## 10.2 대기권 방식

구매는 가능하지만 추천이 준비되면 제공한다.

권장 문구:

> 추가 추천권은 구매되었고, 조건에 맞는 추천이 준비되면 바로 사용할 수 있어요.

장점:

- 결제 전환은 유지된다.

단점:

- 즉시 제공되지 않으면 불만이 생길 수 있다.

## 10.3 권장 방식

초기에는 **구매 전 제한 방식**을 권장한다.

프롤로그는 추천 품질과 신뢰가 핵심이므로, 추천 후보가 부족한 상황에서 무리하게 판매하지 않는 것이 장기적으로 더 안전하다.

# 11. 추천권과 인증 수준의 관계

추가 추천권은 인증을 우회하는 수단이 되어서는 안 된다.

## 11.1 필수 인증 미완료자

필수 인증을 완료하지 않은 사용자는 추가 추천권을 구매할 수 없다.

필수 인증:

- 본인 인증
- 얼굴 인증
- 나이 확인
- 관계 목적 설문
- 싱글 상태 서약
- 매너 서약

## 11.2 강력 권장 인증 완료자

직업/재직 인증, 거주 지역 인증 등을 완료한 사용자는 추천 품질과 추천 우선순위에서 긍정적으로 반영될 수 있다.

## 11.3 고신뢰 추천권

고신뢰 풀 추천권은 별도 조건을 둔다.

- 고신뢰 풀 입장 조건 충족
- Trust Plus 가입 또는 고신뢰 추가 추천권 구매
- 중대한 제재 이력 없음

# 12. 추천권과 거절 사유 데이터

추가 추천권은 추천 개선 루프와 연결되어야 한다.

## 12.1 거절 사유 입력 후 추가 추천권 제안

사용자가 추천을 넘길 때 거절 사유를 입력하면, 다음 추천이 더 정교해진다는 기대를 만들 수 있다.

권장 흐름:

1. 사용자가 추천을 넘긴다.
2. 가장 가까운 거절 사유를 선택한다.
3. “다음 추천에 반영할게요” 문구를 보여준다.
4. 기본 추천이 소진된 경우 추가 추천권을 제안한다.

권장 문구:

> 알려주신 이유는 다음 추천에 반영할게요.  
> 추가 추천권으로 한 사람의 프롤로그를 더 받아볼 수 있어요.

## 12.2 주의점

거절 사유 입력 직후 과도하게 결제를 유도하면 사용자가 피로감을 느낄 수 있다.

따라서 결제 제안은 부드럽고 선택적으로 노출한다.

# 13. 추천권과 공정성 정책

추가 추천권은 특정 사용자를 과도하게 소비하거나, 일부 사용자에게 노출이 몰리는 구조를 만들면 안 된다.

## 13.1 노출 피로 방지

한 사용자가 너무 많은 사람에게 반복 노출되지 않도록 노출 상한을 둔다.

## 13.2 인기 편중 방지

관심을 많이 받는 사용자에게만 추천이 몰리지 않도록 추천 분산 로직을 둔다.

## 13.3 신고/차단 반영

신고, 차단, 제재 이력은 추가 추천권 추천에서도 동일하게 반영한다.

## 13.4 선호 조건 우회 금지

추가 추천권은 사용자의 선호 조건과 상대의 공개/추천 조건을 우회하지 않는다.

# 16. 추천 수 정책 리스크와 대응

## 16.1 리스크: 추천 수가 너무 적어 답답하게 느껴질 수 있음

대응:

- 무료에서도 주 1~2명은 제공한다.
- 추천 소진 시 추가 추천권을 제공한다.
- Plus 전환 이유를 명확히 보여준다.
- 추천 이유와 리포트로 한 명의 가치를 높인다.

## 16.2 리스크: 추가 추천권이 과도한 소비를 유도할 수 있음

대응:

- 월 사용 한도를 둔다.
- 추천권 문구를 “한 사람의 프롤로그를 더 읽어보기”로 표현한다.
- 무제한 묶음 상품을 만들지 않는다.

## 16.3 리스크: 추천 품질이 낮은데 추천권을 판매할 수 있음

대응:

- 추천 후보가 부족하면 구매를 제한한다.
- 추천 품질 기준을 만족하지 못하면 대기 처리한다.
- 추천권 사용 후 만족도와 전환율을 추적한다.

## 16.4 리스크: 유료 사용자가 무료 사용자보다 지나치게 유리해질 수 있음

대응:

- 유료는 추천 빈도와 리포트 깊이를 높이되, 상대 노출 우선권을 과도하게 주지 않는다.
- 인증과 프로필 완성도를 추천 품질의 핵심 요소로 유지한다.
