# 1. 문서 목적

이 문서는 프롤로그의 수익 모델 방향을 정의하기 위한 BM 전략 문서다.

프롤로그는 일반 소개팅 앱처럼 무한 스와이프, 좋아요 판매, 부스트 노출을 중심으로 수익화하는 서비스가 아니다. 프롤로그의 핵심 가치는 더 많은 사람을 보여주는 것이 아니라, 신뢰할 수 있는 사람을 더 납득 가능한 방식으로 추천하는 데 있다.

따라서 BM도 다음 가치와 일관되어야 한다.

1. 인증 기반 신뢰
2. 제한된 큐레이션 추천
3. 이야기형 프로필
4. 추천 이유와 매칭 리포트
5. 고신뢰 사용자 풀
6. 안전하고 성숙한 대화 경험

이 문서는 프롤로그가 어떤 방식으로 돈을 벌고, 어떤 방식으로는 돈을 벌지 않을 것인지를 정리한다.

---

# 2. BM 한 줄 정의

**프롤로그의 BM은 좋아요를 많이 파는 구조가 아니라, 신뢰할 수 있는 큐레이션 추천 경험에 비용을 지불하는 멤버십 구조다.**

더 구체적으로는 다음과 같이 정의한다.

> 무료로 시작할 수 있지만, 더 안정적인 추천 빈도, 상세 매칭 리포트, 고신뢰 풀, 추가 추천권에 비용을 지불하는 인증 기반 큐레이션 멤버십 모델.

# 3. BM 설계 원칙

## 3.1 더 많이 보여주는 것이 아니라 더 납득되게 추천한다

프롤로그는 무한 스와이프 앱이 아니다. 따라서 BM도 무제한 좋아요, 무제한 탐색, 무제한 노출을 중심으로 만들지 않는다.

유료화의 핵심 가치는 다음이다.

- 추천 빈도의 안정성
- 추천 품질 개선
- 추천 이유의 깊이
- 인증 기반 신뢰도
- 고신뢰 풀 접근
- 대화 전환 보조

## 3.2 돈으로 신뢰를 사게 만들지 않는다

프롤로그에서 신뢰는 결제로만 얻는 것이 아니다. 신뢰는 인증, 프로필 완성도, 관계 목적 확인, 매너 이력, 신고/제재 이력 등을 통해 형성된다.

따라서 고신뢰 풀은 결제만으로 입장할 수 없어야 한다.

고신뢰 풀 접근 조건은 다음 두 요소가 결합되어야 한다.

1. 필수 또는 권장 인증 조건 충족
2. 유료 멤버십 또는 고신뢰 풀 이용권

## 3.3 남녀 동일 가격을 기본 원칙으로 한다

프롤로그는 신뢰와 진정성을 강조하는 브랜드다. 따라서 특정 성별에만 과도하게 과금하거나, 남성에게만 결제 압박을 주는 구조는 지양한다.

기본 원칙은 남녀 동일 가격이다.

다만 초기 유저풀 균형을 위해 다음과 같은 운영 혜택은 가능하다.

- 초기 베타 초대권
- 특정 지역/연령대 모집 쿠폰
- 인증 완료자 무료 체험
- 성비 균형을 위한 한시적 프로모션

이 경우에도 기본 가격 구조는 성별 차별이 아니라 운영상 한시 혜택으로 설명해야 한다.

## 3.4 추가 추천권은 명확한 유료 상품으로 포함한다

프롤로그는 추천 수를 제한하는 서비스이기 때문에, 기본 추천을 모두 본 사용자가 더 보고 싶어 하는 니즈가 발생할 수 있다.

따라서 **추가 추천권**은 BM에 명확히 포함한다.

다만 추가 추천권은 다음 원칙을 따른다.

1. 무제한 판매하지 않는다.
2. 추천 품질을 해치지 않는 범위에서만 제공한다.
3. 사용자의 인증 수준과 선호 조건을 우회하지 않는다.
4. 상대에게 더 많이 노출되는 부스트가 아니라, 사용자 본인이 추가 추천을 받는 권리다.
5. 매칭을 보장하지 않는다.
6. 고신뢰 풀 추천은 해당 조건을 충족한 사용자에게만 제공한다.

즉, 추가 추천권은 “사람을 더 많이 소비하는 권리”가 아니라 **큐레이션된 추천을 조금 더 받아보는 권리**로 설계한다.

## 3.5 노출권/부스트 중심 BM은 지양한다

다음 방식은 프롤로그 브랜드와 충돌할 가능성이 높다.

- 프로필 상위 노출권
- 인기 회원에게 우선 노출
- 좋아요 무제한
- 무제한 스와이프
- 랭킹 기반 노출
- 돈을 내면 인증 없이 더 많이 추천받는 구조

프롤로그는 돈을 더 낸 사람이 생태계 안에서 일방적으로 더 강한 힘을 갖는 구조가 아니라, 더 성실하게 인증하고 프로필을 작성한 사람이 더 좋은 추천 경험을 얻는 구조여야 한다.

# 4. 핵심 수익원 구조

프롤로그의 수익원은 다음 5개 축으로 구성한다.

| 수익원 | 우선순위 | 설명 |
|---|---:|---|
| 월 구독 멤버십 | 1 | 정기 추천, 상세 리포트, 대화 보조 기능 제공 |
| 추가 추천권 | 1 | 기본 추천 소진 후 추가 큐레이션 추천 제공 |
| 고신뢰 풀 | 2 | 인증 조건을 충족한 사용자끼리 추천받는 프리미엄 풀 |
| 상세 리포트/프로필 개선 리포트 | 2 | 추천 이유, 프로필 완성도, 대화 주제 등 고도화 |
| 인증 관련 부가 서비스 | 3 | 실제 비용이 발생하는 인증 처리 또는 추가 확인 기능 |

초기 MVP에서는 **무료 플랜 + Plus 멤버십 + 추가 추천권**을 1차 BM으로 검증한다. 고신뢰 풀과 리포트 고도화는 유저풀이 충분히 쌓인 이후 도입한다.

# 5. 무료 플랜 전략

## 5.1 무료 플랜의 목적

무료 플랜의 목적은 수익이 아니라 사용자에게 프롤로그의 핵심 가치를 경험하게 하는 것이다.

무료 사용자는 다음을 경험할 수 있어야 한다.

- 가입
- 본인 인증
- 얼굴 인증
- 관계 목적 설문
- 프로필 작성
- 제한된 추천 카드
- 요약 매칭 리포트
- 관심/거절
- 상호 관심 시 대화

무료에서도 핵심 경험이 너무 막혀 있으면 사용자는 프롤로그의 차이를 이해하지 못한다. 반대로 무료가 너무 넉넉하면 유료 전환 이유가 약해진다.

## 5.2 무료 플랜 권장 범위

| 항목 | 무료 플랜 권장 정책 |
|---|---|
| 기본 가입 | 가능 |
| 필수 인증 | 가능 |
| 프로필 작성 | 가능 |
| 기본 추천 | 제한 제공 |
| 매칭 리포트 | 요약 제공 |
| 관심 보내기 | 제한 제공 |
| 대화 | 상호 관심 시 가능 |
| 추가 추천권 구매 | 가능 |
| 고신뢰 풀 | 불가 또는 체험 제한 |
| 상세 리포트 | 불가 |
| 프로필 개선 리포트 | 불가 또는 미리보기 |

# 6. 유료 멤버십 전략

## 6.1 초기 유료 멤버십: 프롤로그 Plus

초기에는 유료 플랜을 여러 개 만들기보다 **프롤로그 Plus** 하나로 시작하는 것이 좋다.

이유는 다음과 같다.

1. 사용자가 이해하기 쉽다.
2. 어떤 가치에 결제했는지 분석하기 쉽다.
3. 초기 운영 복잡도가 낮다.
4. 고신뢰 풀 도입 전에도 유료 가치를 만들 수 있다.

## 6.2 Plus의 핵심 제공 가치

| 제공 가치 | 설명 |
|---|---|
| 정기 큐레이션 추천 | 무료보다 안정적인 추천 빈도 제공 |
| 상세 매칭 리포트 | 잘 맞는 지점, 차이점, 대화 주제 제공 |
| 추가 추천권 기본 포함 또는 할인 | 월 소량의 추가 추천권 제공 또는 구매 할인 |
| 첫 대화 주제 추천 | 상호 관심 이후 대화 전환 보조 |
| 프로필 완성도 피드백 | 더 좋은 추천을 받기 위한 개선 가이드 |
| 고급 선호 조건 일부 | 추천 품질 개선 목적의 선호 조건 설정 |
| 인증 완료자 우선 추천 | 인증 수준이 높은 사용자에게 더 좋은 추천 경험 제공 |

## 6.3 Plus에서 피해야 할 가치 표현

프롤로그 Plus는 다음처럼 표현하면 안 된다.

- 더 인기 많은 사람을 볼 수 있어요
- 상위 회원에게 노출돼요
- 좋아요를 무제한 보낼 수 있어요
- 프리미엄 회원만의 고스펙 상대를 볼 수 있어요
- 돈을 내면 더 좋은 사람을 만나요

권장 표현은 다음이다.

- 더 안정적인 큐레이션 추천을 받아보세요
- 추천 이유를 더 자세히 확인하세요
- 프로필 완성도를 높여 더 잘 맞는 추천을 받을 수 있어요
- 추가 추천권으로 오늘 더 한 사람의 이야기를 읽어보세요

# 7. 추가 추천권 전략

## 7.1 추가 추천권의 역할

추가 추천권은 프롤로그 BM에서 반드시 포함할 유료 상품이다.

프롤로그는 추천 수를 제한하기 때문에, 사용자는 다음 상황에서 추가 추천권을 필요로 할 수 있다.

- 오늘 받은 추천을 모두 확인했지만 더 보고 싶을 때
- 특정 기간 동안 더 적극적으로 만남을 찾고 싶을 때
- 무료 플랜에서 프롤로그의 가치를 더 확인해보고 싶을 때
- Plus 사용자지만 이번 주 추천을 더 받고 싶을 때
- 거절 사유 입력 후 더 맞는 추천을 바로 보고 싶을 때

## 7.2 추가 추천권의 정의

**추가 추천권은 기본 제공 추천을 모두 본 사용자가, 프롤로그의 추천 로직에 따라 추가 추천 카드 1개를 열람할 수 있는 권리다.**

추가 추천권은 다음을 의미하지 않는다.

- 매칭 보장
- 대화 보장
- 인기 사용자 노출 보장
- 상대에게 우선 노출
- 고신뢰 풀 자동 접근
- 인증 조건 우회

## 7.3 추가 추천권 상품 구성 가설

| 상품 | 가격 가설 | 설명 |
|---|---:|---|
| 추가 추천권 1장 | 2,900~3,900원 | 한 명의 추가 추천 카드 열람 |
| 추가 추천권 3장 | 7,900~9,900원 | 단기 적극 이용자용 |
| 추가 추천권 5장 | 12,900~14,900원 | 월 제한 내 묶음 구매 |
| Plus 월 포함권 | 월 1~3장 | 구독 가치 강화용 |
| Trust Plus 월 포함권 | 월 3~5장 | 고신뢰 풀 사용자용 |

초기에는 1장과 3장 묶음만 운영하고, 5장 묶음은 추천 품질과 유저풀 규모가 안정된 뒤 도입하는 것이 안전하다.

## 7.4 추가 추천권 제한 정책

추가 추천권은 무제한 판매하지 않는다.

권장 제한은 다음과 같다.

| 사용자 유형 | 월 구매/사용 제한 가설 |
|---|---:|
| 무료 사용자 | 월 3장까지 |
| Plus 사용자 | 월 5장까지 |
| Trust Plus 사용자 | 월 5~7장까지 |
| 인증 미완료 사용자 | 구매 불가 |
| 제재 이력 사용자 | 구매 제한 또는 불가 |

이 제한은 프롤로그의 큐레이션 가치를 유지하고, 과도한 탐색형 사용을 막기 위한 장치다.

# 8. 고신뢰 풀 전략

## 8.1 고신뢰 풀의 목적

고신뢰 풀은 프롤로그의 장기 핵심 수익원이다.

고신뢰 풀은 단순히 돈을 더 낸 사람들이 모이는 곳이 아니라, 인증과 매너 기준을 충족한 사용자들이 더 신뢰도 높은 추천을 받을 수 있는 공간이다.

## 8.2 고신뢰 풀 입장 조건 가설

| 조건 | 기준 |
|---|---|
| 본인 인증 | 필수 |
| 얼굴 인증 | 필수 |
| 관계 목적 설문 | 필수 |
| 싱글 상태 서약 | 필수 |
| 매너 서약 | 필수 |
| 직업/재직 인증 | 필수 또는 강력 권장 |
| 거주 지역 인증 | 권장 |
| 신고/제재 이력 | 중대 이력 없어야 함 |
| 유료 멤버십 | 필요 |

## 8.3 고신뢰 풀 수익화 방식

| 방식 | 설명 |
|---|---|
| Trust Plus 멤버십 | 고신뢰 풀 접근 포함 월 구독 |
| 고신뢰 추가 추천권 | 조건 충족자에게만 판매 |
| 상세 신뢰 리포트 | 인증 상태와 관계 성향 근거를 더 자세히 제공 |
| 인증 완료자 우선 추천 | 인증 수준 높은 사용자끼리 추천 우선 |

# 9. 리포트 기반 수익화

프롤로그의 차별점은 추천 이유와 매칭 리포트다. 따라서 리포트는 유료 가치로 확장할 수 있다.

## 9.1 매칭 리포트

무료 사용자는 요약 리포트를 보고, Plus 사용자는 상세 리포트를 볼 수 있다.

상세 리포트 포함 가능 항목:

- 잘 맞는 관계 성향
- 공통 라이프스타일
- 대화 스타일 유사점
- 확인하면 좋은 차이점
- 첫 대화 추천 주제
- 추천 근거 요약
- 인증 기반 신뢰 정보 요약

## 9.2 프로필 개선 리포트

프로필 개선 리포트는 특히 남성 사용자에게 지불 가치가 있을 수 있다.

포함 가능 항목:

- 프로필 완성도
- 부족한 문답 항목
- 사진 구성 가이드
- 관계 목적 표현 가이드
- 추천에서 불리할 수 있는 정보 부족 영역
- 더 좋은 추천을 받기 위한 개선 제안

단, 프로필 개선 리포트는 외모 평가나 스펙 평가처럼 느껴지지 않아야 한다.

# 12. BM 리스크와 대응

## 12.1 리스크: 추가 추천권이 무한 스와이프처럼 느껴질 수 있음

대응:

- 월 사용 제한을 둔다.
- “더 많은 사람 보기”가 아니라 “한 사람의 프롤로그를 더 읽어보기”로 표현한다.
- 추천권 사용 후에도 추천 이유와 리포트를 제공한다.
- 추천 품질이 부족한 경우 판매하지 않거나 대기 처리한다.

## 12.2 리스크: 유료 결제가 신뢰를 사는 것처럼 보일 수 있음

대응:

- 인증 조건과 결제를 분리한다.
- 고신뢰 풀은 인증 조건 충족자를 대상으로 한다.
- 돈만 내면 고신뢰 사용자가 되는 구조를 금지한다.

## 12.3 리스크: 가격이 결정사처럼 무겁게 느껴질 수 있음

대응:

- 초기 Plus 가격은 월 1만 원대 후반 중심으로 검증한다.
- 고가 가입비, 장기 계약, 환불 복잡성을 피한다.
- 월 단위로 가볍게 시작할 수 있게 한다.

## 12.4 리스크: 무료 사용자가 너무 적은 경험만 얻을 수 있음

대응:

- 무료에서도 첫 추천과 요약 리포트를 경험하게 한다.
- 추가 추천권을 통해 무료 사용자도 제한적으로 더 경험할 수 있게 한다.
- 과도한 잠금 화면을 줄인다.

# 13. 현재 기준 권장 BM 구조

초기 권장 구조는 다음과 같다.

| 구분 | 무료 | Plus | Trust Plus |
|---|---:|---:|---:|
| 가격 | 0원 | 월 14,900~19,900원 | 월 29,000~39,000원 |
| 기본 추천 | 제한 제공 | 정기 제공 | 고신뢰 풀 포함 제공 |
| 매칭 리포트 | 요약 | 상세 | 상세+신뢰 근거 강화 |
| 추가 추천권 구매 | 가능 | 가능+할인 또는 월 포함 | 가능+월 포함 |
| 추가 추천권 월 한도 | 낮음 | 중간 | 중간~높음 |
| 프로필 피드백 | 제한 | 제공 | 제공 |
| 첫 대화 주제 | 제한 | 제공 | 제공 |
| 고신뢰 풀 | 불가 | 일부 체험 가능 | 가능 |

MVP에서는 **무료 + Plus + 추가 추천권**을 먼저 출시하고, Trust Plus는 유저풀과 인증률이 안정된 뒤 도입한다.
