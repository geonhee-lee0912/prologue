# 1. 문서 목적

이 문서는 프롤로그 서비스 안에서 사용할 공식 용어와 의미를 정리하는 용어 사전이다.

프롤로그는 인증 기반 소개팅 앱이면서 책방/서점 메타포를 사용하는 브랜드다. 따라서 같은 기능도 일반적인 앱 용어와 프롤로그식 표현이 함께 존재할 수 있다. 용어 기준이 없으면 화면마다 표현이 달라지고, 사용자가 기능을 이해하기 어려워질 수 있다.

이 문서는 다음 목적을 가진다.

1. 앱 내부 공식 용어를 정리한다.
2. 일반 기능명과 프롤로그식 표현의 관계를 정의한다.
3. 사용 가능한 용어와 사용 금지 용어를 분리한다.
4. 기획, 디자인, 개발, 마케팅 문서에서 같은 용어를 사용하게 한다.
5. 향후 IA, 데이터 항목 정의서, 기능요구사항서 작성의 기준을 만든다.

# 2. 용어 운영 원칙

## 2.1 사용자에게는 직관성과 브랜드감을 함께 제공한다

프롤로그식 표현은 브랜드 차별화에 도움이 되지만, 기능 이해를 방해해서는 안 된다. 따라서 핵심 행동 버튼은 직관적 표현을 우선하고, 브랜드 표현은 제목, 보조 문구, 섹션명에 사용한다.

예:

- 제목: 첫문장이 이어졌어요
- 버튼: 대화 시작하기

## 2.2 내부 문서와 개발 용어는 명확하게 쓴다

개발, 정책, 운영 문서에서는 은유보다 기능적 용어를 우선한다.

예:

- 내부 용어: 추천 카드
- 사용자 노출 표현: 오늘의 프롤로그
- 데이터명 예시: recommendation_card

## 2.3 하나의 기능에 너무 많은 이름을 붙이지 않는다

브랜드 표현이 늘어나면 서비스가 감성적이지만 복잡하게 느껴질 수 있다. 공식 용어와 보조 표현을 구분한다.

## 2.4 사람을 상품화하는 용어는 사용하지 않는다

책방/서점 메타포를 사용하더라도 대출, 반납, 품절, 장바구니, 소장 같은 표현은 사용하지 않는다.

- 혼인상태 인증: 초기 외부 표현으로 사용하지 않음
- 소득 인증: 직접적이고 민감하므로 “경제 안정성 인증”을 우선 검토
- 스펙 인증: 사람을 조건화하는 느낌이 강해 사용 금지
- 고급 회원 인증: 등급화 느낌이 강해 사용 금지

| 합격/불합격 | 사용자 평가 느낌 | 인증 완료/재시도 필요 |
| 등급 | 사람 줄 세우기 | 신뢰 배지, 인증 상태 |
| 고스펙 | 조건 중심 | 직업/재직 인증, 신뢰 정보 |
| 대출/반납 | 사람 상품화 | 관심 남기기/넘기기 |
| 품절/재고 | 사람 상품화 | 현재 추천 불가 |
| 운명의 상대 | 과장 | 대화를 시작해볼 만한 상대 |
| 완벽한 매칭 | 보장 | 잘 맞을 수 있는 지점 |

# 10. 개발/기획 문서 작성 시 명명 규칙

## 10.1 파일명/문서명

문서명은 다음 구조를 따른다.

> 영역번호_프롤로그_문서명

예:

- 03_프롤로그 사용자 여정 설계서
- 04_프롤로그 인증 정책 기획서
- 06_프롤로그 랜딩페이지 기획서

## 10.2 화면명

화면명은 기능 중심으로 명확히 쓴다.

권장:

- 홈
- 추천 카드 상세
- 매칭 리포트
- 프로필 작성
- 관계 목적 설문
- 대화방
- 신고하기

비권장:

- 오늘의 책방
- 이야기방
- 첫문장실
- 서재 입구

브랜드 표현은 화면 내부 섹션명이나 카피로 사용한다.

## 10.3 데이터 항목명

개발 문서에서는 영어 스네이크 케이스를 사용한다.

예:

- user_id
- profile_id
- verification_status
- relationship_intent
- recommendation_id
- pass_reason
- mutual_interest_status
- chat_room_id
