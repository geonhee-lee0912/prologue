# 1. 문서 목적

이 문서는 프롤로그에서 수집하는 개인정보와 프로필 정보, 인증 정보, 추천/매칭 정보, 대화 정보의 공개 범위와 보관 원칙을 정의하기 위한 정책 문서다.

프롤로그는 인증 기반 진지한 소개팅 앱이므로, 사용자의 신뢰를 얻기 위해 본인 인증, 얼굴 인증, 관계 목적, 직업/재직 인증, 싱글 상태 서약 등 민감할 수 있는 정보를 다룬다. 따라서 어떤 정보를 수집하고, 어떤 정보가 상대에게 보이며, 어떤 정보는 추천 알고리즘에만 사용되고, 어떤 정보는 저장하지 않거나 최소한으로 보관하는지 명확히 해야 한다.

이 문서는 다음 의사결정의 기준으로 활용한다.

1. 필수 정보와 선택 정보를 분리한다.
2. 프로필에 공개되는 정보와 비공개 정보를 구분한다.
3. 인증 원본과 인증 결과의 저장/공개 방식을 정의한다.
4. 추천 알고리즘에 사용되는 정보와 사용자에게 노출되는 정보를 구분한다.
5. 단계별 공개 범위를 설계한다.
6. 사용자가 자신의 정보 공개를 통제할 수 있도록 기준을 만든다.
7. 개인정보 처리방침, 이용약관, 동의 화면, 온보딩 문구의 기준을 만든다.

이 문서는 법률 자문을 대체하지 않는다. 실제 출시 전 개인정보보호법, 위치정보, 전자상거래, 통신비밀, 결제 정보, 민감정보 처리 가능성, 국외 이전 여부 등에 대한 법무 검토가 필요하다.

# 2. 정책 한 줄 정의

**프롤로그의 개인정보_공개범위 정책은 사용자가 신뢰 가능한 추천을 받기 위해 필요한 정보는 제공하되, 상대에게 공개되는 정보는 사용자가 이해하고 통제할 수 있게 하는 기준이다.**

사용자에게 전달할 핵심 메시지는 다음과 같다.

> 신뢰를 위한 정보는 신중하게 확인하고, 공개는 필요한 만큼만 합니다.

# 3. 기본 원칙

## 3.1 필수 수집은 최소화한다

서비스 제공에 꼭 필요한 정보만 필수로 수집한다. 사용자가 민감하게 느낄 수 있는 정보는 선택 입력 또는 선택 인증으로 분리한다.

## 3.2 인증 원본보다 인증 결과를 사용한다

신분증, 얼굴 인증 이미지, 재직 증빙, 소득 증빙 등 원본 정보는 가능하면 저장을 최소화하고, 서비스 화면에는 인증 결과와 배지만 표시한다.

## 3.3 공개 범위는 사용자가 이해할 수 있어야 한다

사용자는 어떤 정보가 누구에게 보이는지 쉽게 이해해야 한다. 온보딩, 프로필 작성, 인증 단계에서 정보 사용 목적과 공개 범위를 짧고 명확하게 안내한다.

## 3.4 추천에 쓰는 정보와 상대에게 보이는 정보는 다를 수 있다

일부 정보는 추천 품질을 높이기 위해 사용되지만, 상대에게 그대로 공개되지 않을 수 있다. 예를 들어 관계 목적, 연락 빈도, 선호 생활권은 추천에는 쓰이되 요약 형태로만 노출할 수 있다.

## 3.5 안전을 위해 단계적 공개를 적용한다

처음부터 모든 정보를 공개하지 않는다. 추천 단계, 상호 관심 단계, 대화 단계, 사용자가 직접 공개하는 단계에 따라 공개 범위를 달리한다.

## 3.6 삭제와 탈퇴 기준을 명확히 한다

사용자는 탈퇴 시 자신의 정보가 어떻게 삭제되는지 알아야 한다. 다만 신고/제재 기록 등 서비스 안전을 위해 일정 기간 보관이 필요한 정보는 별도 기준을 둔다.

> 휴대폰 번호와 계정 정보는 본인 확인과 계정 보호를 위해 사용되며, 다른 사용자에게 공개되지 않습니다.

## 5.2 기본 프로필 정보

| 항목 | 수집 목적 | 상대 공개 | 공개 방식 | 비고 |

| 성별 | 추천, 서비스 운영 | 제한 공개 | 추천 대상에 반영 | 공개 방식 검토 |
| 거주 지역 | 생활권 추천 | 공개 | 시/구 또는 권역 | 상세 주소 금지 |
| 직업군 | 신뢰, 추천 | 공개 | 직업군/업계 중심 | 회사명은 선택 |
| 학력 | 선택 신뢰 정보 | 선택 공개 | 학교명/학위 여부 검토 | 민감도 주의 |
| 키/체형 | 선택 프로필 | 선택 공개 | 사용자 입력 시 | 필수화 지양 |
| 흡연/음주 | 라이프스타일 추천 | 공개 가능 | 선택형 응답 | 추천에 중요 |

### 공개 원칙

- 거주지는 상세 주소가 아니라 생활권 중심으로 보여준다.
- 직업은 정확한 회사명보다 직업군 또는 업계 중심 노출을 기본으로 한다.
- 회사명, 학교명 등 특정 가능한 정보는 선택 공개로 둔다.

## 5.3 사진 정보

사진은 소개팅 서비스에서 매칭에 큰 영향을 주지만, 동시에 사용자가 가장 민감하게 느끼는 정보다.

| 항목 | 수집 목적 | 상대 공개 | 공개 방식 | 비고 |
|---|---|---|---|---|
| 대표 사진 | 프로필 신뢰, 호감 판단 | 공개 | 추천 카드/상세 | 기본 필요 |
| 일상 사진 | 취향/분위기 전달 | 선택 공개 | 프로필 상세 | 사용자가 선택 |
| 얼굴 인증 이미지 | 도용 방지 | 비공개 | 인증 결과만 배지 | 원본 노출 금지 |
| 블러 처리 사진 | 초기 안전 장치 | 선택 가능 | 단계적 공개 | 여성 안전감 검토 |

## 5.3.1 사진 공개 정책 후보

### 안 A: 추천 단계에서 대표 사진 공개

- 장점: 사용자가 빠르게 판단 가능
- 단점: 기존 앱처럼 사진 중심으로 흐를 수 있음
- 적합성: MVP 구현이 단순함

### 안 B: 대표 사진은 공개하되, 추가 사진은 상호 관심 후 공개

- 장점: 기본 호감 판단과 정보 통제의 균형
- 단점: 일부 사용자에게 답답할 수 있음
- 적합성: 프롤로그의 안전감과 신중함에 적합

### 안 C: 사용자가 사진별 공개 단계를 설정

- 장점: 통제감이 높음
- 단점: UX와 개발 복잡도 증가
- 적합성: MVP 이후 고도화 방향

### 현재 권장 방향

초기 MVP에서는 **대표 사진은 추천 카드에서 공개하고, 추가 사진은 프로필 상세 또는 상호 관심 단계에서 공개 범위를 선택할 수 있게 하는 방향**을 우선 검토한다.

## 5.5 관계 목적 정보

관계 목적 정보는 프롤로그 추천의 핵심 입력값이지만, 너무 직접적으로 공개하면 부담을 줄 수 있다.

| 항목 | 추천 사용 | 상대 공개 | 공개 방식 |
|---|---|---|---|
| 원하는 만남 | 사용 | 공개 가능 | “진지한 연애”, “장기 관계 가능성” 등 요약 |
| 관계 시작 속도 | 사용 | 공개 가능 | “천천히 알아가는 편” |
| 연락 빈도 선호 | 사용 | 공개 가능 | 요약 문구 |
| 장기 관계 의향 | 사용 | 제한 공개 | 직접 표현보다 요약 |
| 결혼 가능성 응답 | 사용 가능 | 기본 비공개 또는 요약 | 결혼중개 오인 주의 |

### 공개 원칙

- 결혼 관련 응답을 직접 노출하기보다 “장기적인 관계 가능성을 열어둔 편”처럼 부드럽게 표현한다.
- 사용자가 공개를 원하지 않는 세부 응답은 매칭 리포트에서 요약 수준으로만 다룬다.

| 대화 내용 | 대화 상대에게 공개 | 신고/분쟁 시 제한 접근 | 일반 열람 지양 |
| 신고 내용 | 상대에게 비공개 | 운영 검토 | 신고자 보호 |
| 차단 여부 | 직접 고지 안 함 | 내부 기록 | 상대에게 사유 미노출 |
| 제재 기록 | 상대에게 비공개 | 내부 관리 | 안전 운영 |
| 노쇼/비매너 기록 | 상대에게 비공개 | 내부/추천 반영 검토 | 평판화 주의 |

# 6. 단계별 공개 범위

프롤로그는 추천 단계부터 모든 정보를 공개하지 않고, 관계 진행 단계에 따라 공개 범위를 다르게 설계한다.

## 6.1 단계 정의

| 단계 | 설명 |
|---|---|
| 가입/온보딩 | 본인 확인, 인증, 프로필 작성 단계 |
| 추천 전 | 추천 알고리즘이 사용자 정보를 분석하는 단계 |
| 추천 카드 | 상대에게 처음 노출되는 단계 |
| 프로필 상세 | 상대가 더 자세히 읽는 단계 |
| 상호 관심 | 서로 관심을 표현한 단계 |
| 대화 | 직접 메시지를 주고받는 단계 |
| 만남 이후 | 신고/피드백/차단 가능 단계 |

## 6.2 단계별 공개 정책

| 정보 | 추천 카드 | 프로필 상세 | 상호 관심 후 | 대화 중 |
|---|---|---|---|---|
| 대표 사진 | 공개 | 공개 | 공개 | 공개 |
| 추가 사진 | 제한/선택 | 공개/선택 | 공개 가능 | 공개 가능 |
| 나이 | 공개 | 공개 | 공개 | 공개 |
| 지역 | 권역 공개 | 시/구 공개 가능 | 동일 | 사용자 선택 |
| 직업군 | 공개 | 공개 | 공개 | 사용자 선택 |
| 회사명 | 기본 비공개 | 선택 공개 | 선택 공개 | 직접 공유 가능 |
| 인증 배지 | 공개 | 공개 | 공개 | 공개 |
| 관계 목적 요약 | 공개 | 공개 | 공개 | 대화 가능 |
| 세부 관계 문답 | 일부 공개 | 공개/선택 | 공개 가능 | 직접 대화 |
| 연락처/SNS | 비공개 | 비공개 | 비공개 | 사용자가 직접 공유 |
| 거절 사유 | 비공개 | 비공개 | 비공개 | 비공개 |

# 7. 공개 범위 설정 UX 원칙

## 7.1 설정은 너무 복잡하지 않아야 한다

모든 항목마다 공개/비공개를 세세하게 설정하게 하면 사용자는 피로감을 느낀다. MVP에서는 핵심 민감 정보 중심으로 설정한다.

MVP 공개 범위 설정 후보:

1. 추가 사진 공개 단계
2. 회사명/학교명 공개 여부
3. 상세 지역 공개 여부
4. 민감 문답 공개 여부
5. 프로필 검색/추천 노출 여부

## 7.2 공개 전 미리보기 제공

사용자는 상대에게 내 프로필이 어떻게 보이는지 확인할 수 있어야 한다.

권장 기능:

- “상대에게 보이는 내 프로필 미리보기”
- 공개/비공개 표시
- 인증 배지 노출 방식 확인

## 7.3 민감 정보 입력 시 경고

사용자가 자기소개나 문답에 연락처, SNS, 회사명, 주소 등 민감한 정보를 입력하면 안내한다.

예시:

> 연락처나 상세 주소처럼 민감한 정보는 프로필에 공개하지 않는 것을 권장합니다.

# 8. 정보 사용 목적 안내

사용자에게 정보 사용 목적을 명확히 안내해야 한다.

## 8.1 인증 단계

> 이 정보는 본인 확인과 허위 프로필 방지를 위해 사용되며, 다른 사용자에게 원본이 공개되지 않습니다.

## 8.2 관계 목적 설문

> 답변은 더 잘 맞는 사람을 추천하기 위해 사용됩니다. 일부 내용은 프로필과 매칭 리포트에 요약되어 표시될 수 있습니다.

## 8.3 거절 사유 입력

> 이 내용은 상대에게 공개되지 않으며, 추천 품질 개선을 위해서만 사용됩니다.

## 8.4 신고

> 신고 내용은 상대에게 공개되지 않으며, 프롤로그 운영 기준에 따라 검토됩니다.

## 8.5 탈퇴

> 탈퇴하면 프로필과 추천 정보는 삭제됩니다. 다만 서비스 안전과 분쟁 대응을 위해 필요한 기록은 관련 기준에 따라 일정 기간 보관될 수 있습니다.
