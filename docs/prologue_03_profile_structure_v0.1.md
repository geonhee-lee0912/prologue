# 1. 문서 목적

이 문서는 프롤로그의 이야기형 프로필 구조와 작성 항목, 공개 방식, 추천 활용 방식을 정의하기 위한 문서다.

프롤로그의 프로필은 단순 조건표가 아니다. 기존 소개팅 앱처럼 사진, 나이, 지역, 직업 몇 줄만으로 빠르게 판단하게 만드는 구조를 지양한다. 동시에 너무 긴 자기소개를 요구해 사용자가 부담을 느끼게 해서도 안 된다.

프롤로그의 프로필은 다음 세 가지 목적을 동시에 달성해야 한다.

1. 상대가 어떤 사람인지 이해할 수 있게 한다.
2. 추천 알고리즘이 관계 성향과 라이프스타일을 판단할 수 있게 한다.
3. 사용자가 자신을 외모와 조건만이 아니라 이야기와 태도로 표현할 수 있게 한다.

---

# 2. 프로필 한 줄 정의

**프롤로그의 프로필은 사진과 조건만으로 사람을 판단하지 않도록, 기본 정보, 신뢰 배지, 자기 이야기, 관계 성향, 라이프스타일을 함께 보여주는 이야기형 프로필이다.**

브랜드 관점에서는 다음 문장으로 설명할 수 있다.

> 사람은 조건표가 아니라, 하나의 이야기다.

# 3. 프로필 설계 원칙

## 3.1 조건보다 이야기를 먼저 보조한다

나이, 지역, 직업군 등 기본 정보는 필요하지만, 프로필의 중심은 그 사람이 어떤 일상과 관계 태도를 가진 사람인지 보여주는 것이다.

## 3.2 작성 부담을 낮춘다

이야기형 프로필이라고 해서 긴 글을 강제하지 않는다. 짧은 문장형 질문, 선택형 키워드, 다중 선택형 문항, 자유 입력을 적절히 조합한다.

## 3.3 신뢰 정보는 배지로 부드럽게 보여준다

인증 원본이나 민감 정보를 그대로 보여주지 않는다. 인증 완료 여부와 요약 정보 중심으로 보여준다.

## 3.4 추천 알고리즘과 연결한다

프로필 항목은 단순 표시용이 아니라 추천 품질에 반영되어야 한다. 관계 목적, 생활 리듬, 대화 스타일, 가치관, 취향 키워드는 매칭 리포트의 근거가 된다.

## 3.5 공개 범위를 통제할 수 있게 한다

사진, 직업, 상세 지역, 인증 배지, 민감 답변 등은 공개 범위를 분리할 수 있어야 한다. MVP에서는 최소 공개 정책으로 시작하더라도 향후 확장 가능성을 고려한다.

# 4. 프로필 전체 구조

프롤로그 프로필은 다음 7개 영역으로 구성한다.

| 영역 | 프롤로그식 명칭 | 목적 | MVP 우선순위 |
|---|---|---|---:|
| 1. 사진 | 사진 | 첫인상과 신뢰 형성 | 1 |
| 2. 기본 정보 | 기본 정보 | 나이, 지역, 직업군 등 최소 이해 | 1 |
| 3. 신뢰 배지 | 신뢰 배지 | 인증 완료 상태 표시 | 1 |
| 4. 자기소개 | 나의 프롤로그 | 한 사람의 분위기와 태도 표현 | 1 |
| 5. 상세 문답 | 이야기의 목차 | 일상, 취향, 가치관 이해 | 1 |
| 6. 관계 성향 | 관계의 문체 | 관계 속도, 연락, 대화 스타일 | 1 |
| 7. 대화 단서 | 첫 대화 키워드 | 대화 시작을 돕는 소재 | 2 |

# 5. 영역별 상세 구조

## 5.1 사진

### 목적

상대가 사용자의 실제 분위기와 인상을 이해할 수 있도록 돕는다.

### 구성

- 대표 사진 1장: 필수
- 추가 사진 2~4장: 권장
- 일상/취향 사진: 선택

### 사진 유형

| 유형 | 설명 | 권장 여부 |
|---|---|---|
| 얼굴이 잘 보이는 사진 | 기본 신뢰 형성 | 필수 |
| 전신 또는 분위기 사진 | 전체 인상 이해 | 권장 |
| 취향이 드러나는 사진 | 대화 소재 제공 | 권장 |
| 활동/여행/취미 사진 | 라이프스타일 이해 | 선택 |

### 사진 가이드

권장:

- 최근 1년 이내 사진
- 얼굴이 분명히 보이는 사진
- 과도한 보정이 없는 사진
- 혼자 나온 사진
- 일상과 취향이 드러나는 사진

비권장:

- 얼굴이 거의 보이지 않는 사진
- 단체 사진만 등록
- 과도한 보정 또는 필터
- 타인 사진 또는 유명인 이미지
- 과도한 노출
- 공격적이거나 불쾌한 이미지

### 공개 정책 고려

사진 공개 범위는 개인정보/공개범위 정책서에서 확정한다. 초기 MVP에서는 추천 카드와 상세 프로필에서 사진을 제공하되, 추후 단계적 공개나 블러 처리 옵션을 검토할 수 있다.

---

## 5.2 기본 정보

### 목적

상대가 추천 카드에서 최소한의 맥락을 빠르게 이해하게 한다.

### MVP 필수 항목

| 항목 | 표시 방식 | 비고 |
|---|---|---|
| 나이 | 만 나이 또는 연령 | 생년월일 원본 비공개 |
| 지역 | 시/구 또는 생활권 | 상세 주소 비공개 |
| 직업군 | 직업 카테고리 | 회사명은 선택 공개 |
| 선호 관계 목적 | 요약 표시 | 상세 답변은 제한 공개 |
| 흡연 여부 | 선택 또는 필수 검토 | 민감도 낮음 |
| 음주 성향 | 선택 또는 필수 검토 | 라이프스타일 매칭 |

### 선택 항목

- 키
- 종교
- 학력
- 반려동물 여부
- 주거 형태
- 자녀 여부
- 결혼 경험 여부

위 항목은 민감하거나 결정사 느낌을 강화할 수 있으므로 MVP 필수 여부를 신중히 검토한다.

---

## 5.3 신뢰 배지

### 목적

상대가 프로필의 기본 신뢰도를 확인할 수 있게 한다.

### 배지 후보

| 배지명 | 의미 | 표시 우선순위 |
|---|---|---:|
| 본인 인증 완료 | 실명 기반 계정 확인 | 1 |
| 얼굴 인증 완료 | 프로필 사진 도용 방지 | 1 |
| 나이 확인 완료 | 타깃 연령 확인 | 1 |
| 관계 목적 확인 | 관계 설문 완료 | 1 |
| 싱글 상태 서약 완료 | 현재 새로운 관계 가능 상태 서약 | 1 |
| 직업/재직 인증 완료 | 직업 정보 신뢰 강화 | 2 |
| 거주 지역 확인 | 실제 생활권 신뢰 강화 | 2 |
| 학력 인증 완료 | 선택 신뢰 정보 | 3 |
| 경제 안정성 인증 | 선택/프리미엄 신뢰 정보 | 3 |

### 표시 원칙

- 배지는 과도하게 등급화하지 않는다.
- 배지 개수가 많다고 사람의 가치가 높아 보이게 만들지 않는다.
- 원본 정보보다 인증 상태를 중심으로 표시한다.
- 민감한 인증은 사용자가 공개 여부를 선택할 수 있어야 한다.

## 5.4 나의 프롤로그

### 목적

사용자가 자신을 한두 문장으로 소개하는 핵심 자기소개 영역이다.

### 작성 방식

- 짧은 문장형 입력
- 예시 제공
- 최소 글자 수는 낮게 설정
- 과도한 꾸밈보다 자연스러운 표현 권장

### 필수 문항 후보

1. 나를 가장 잘 설명하는 한 문장
2. 요즘 가장 즐겁게 하는 일
3. 좋은 관계에서 중요하게 생각하는 것

### 예시 답변 가이드

권장 예시:

- 평일에는 차분히 일하고, 주말에는 산책과 맛있는 커피로 회복하는 편이에요.
- 대화가 잘 통하고 서로의 생활을 존중하는 관계를 좋아해요.
- 새로운 곳을 찾아다니기보다 익숙한 공간에서 편안함을 느끼는 편이에요.

비권장 예시:

- 연락 주세요.
- 나중에 쓸게요.
- 외모 자신 있음.
- 조건 좋은 사람만.

### 운영 정책

- 비속어, 혐오 표현, 성적 표현, 금전 요구, 외부 연락처는 제한한다.
- 허위 정보나 과장 표현은 신고 대상이 될 수 있다.

## 5.5 이야기의 목차

### 목적

상대가 사용자의 일상, 취향, 가치관을 구조적으로 이해하게 한다.

### 카테고리

| 카테고리 | 질문 방향 | 추천 활용 |
|---|---|---|
| 일과 커리어 | 일에 대한 태도, 평일 리듬 | 생활 리듬 매칭 |
| 주말과 휴식 | 쉬는 방식, 외출 선호 | 라이프스타일 매칭 |
| 취향과 관심사 | 음식, 여행, 문화, 운동 | 대화 주제 생성 |
| 친구와 가족 | 관계망과 시간 배분 | 관계 가치관 이해 |
| 대화와 연락 | 대화 빈도와 방식 | 대화 스타일 매칭 |
| 관계 태도 | 관계 시작 속도, 갈등 해결 | 관계 성향 매칭 |

### 문항 예시

#### 일과 커리어

- 평일의 나는 주로 어떤 리듬으로 지내나요?
- 일에서 중요하게 생각하는 것은 무엇인가요?
- 바쁜 시기에는 관계에서 어떤 배려가 필요하다고 느끼나요?

#### 주말과 휴식

- 쉬는 날에는 어떤 시간을 좋아하나요?
- 집에서 쉬는 편인가요, 밖으로 나가는 편인가요?
- 함께하는 주말에서 중요하게 생각하는 것은 무엇인가요?

#### 취향과 관심사

- 요즘 자주 하는 취미나 관심사는 무엇인가요?
- 대화가 잘 이어지는 주제는 무엇인가요?
- 최근 좋았던 장소나 경험이 있나요?

#### 관계 태도

- 좋은 관계에서 가장 중요하게 생각하는 것은 무엇인가요?
- 갈등이 생겼을 때 어떤 방식이 편한가요?
- 서로의 개인 시간은 어느 정도 필요하다고 느끼나요?

## 5.6 관계의 문체

### 목적

관계에서의 속도, 연락 방식, 애정 표현, 갈등 해결 방식을 보여준다.

### 구조화 항목

| 항목 | 선택지 예시 | 추천 활용 |
|---|---|---|
| 관계 시작 속도 | 천천히 / 자연스럽게 / 확신이 들면 빠르게 | 관계 목적 매칭 |
| 연락 빈도 | 부담 없이 / 꾸준히 / 자주 | 대화 스타일 매칭 |
| 대화 스타일 | 깊은 대화 / 가벼운 일상 공유 / 유머 중심 | 첫 대화 추천 |
| 애정 표현 | 말 / 행동 / 시간 / 선물 / 스킨십 | 관계 성향 이해 |
| 갈등 해결 | 시간을 두고 / 바로 대화 / 정리 후 대화 | 리스크 안내 |
| 독립성/친밀감 | 개인 시간 중요 / 함께하는 시간 중요 / 균형 | 관계 지속 가능성 |

### 표시 방식

추천 카드에서는 2~3개 핵심 키워드만 보여주고, 상세 프로필과 매칭 리포트에서 자세히 설명한다.

예시:

- 관계 시작: 천천히 알아가는 편
- 연락: 하루 한두 번 꾸준히
- 갈등 해결: 감정을 정리한 뒤 대화

## 5.7 라이프스타일 키워드

### 목적

상대와의 생활 리듬, 취향, 대화 소재를 빠르게 파악하게 한다.

### 키워드 후보

#### 생활 리듬

- 아침형
- 저녁형
- 주말 외출형
- 집에서 충전형
- 계획형
- 즉흥형

#### 취향

- 카페
- 산책
- 전시
- 영화
- 독서
- 운동
- 맛집
- 여행
- 음악
- 반려동물

#### 관계 분위기

- 차분한 대화
- 유머 있는 대화
- 깊은 대화
- 일상 공유
- 서로의 시간 존중
- 함께 성장

### 사용 방식

- 프로필 요약
- 추천 알고리즘 입력값
- 매칭 리포트의 공통점
- 첫 대화 추천 주제

# 6. 프로필 작성 흐름

MVP 작성 흐름은 다음과 같다.

> 사진 등록 → 기본 정보 입력 → 관계 목적/관계의 문체 선택 → 나의 프롤로그 작성 → 이야기의 목차 문항 선택 작성 → 공개 미리보기 → 완료

## 6.1 필수 작성 최소안

1. 대표 사진 1장
2. 나이/지역/직업군
3. 관계 목적
4. 관계 시작 속도
5. 연락 빈도
6. 나를 설명하는 한 문장
7. 좋은 관계에서 중요하게 생각하는 것
8. 라이프스타일 키워드 3개 이상

## 6.2 권장 작성안

1. 대표 사진 + 추가 사진 2장 이상
2. 기본 정보 완성
3. 관계의 문체 전체 문항 완료
4. 나의 프롤로그 3문항 완료
5. 이야기의 목차 5문항 이상 완료
6. 직업/재직 인증 완료
7. 취향 키워드 5개 이상 선택

# 7. 프로필 완성도

프로필 완성도는 사용자가 좋은 추천을 받기 위해 정보를 얼마나 충분히 입력했는지 보여주는 지표다.

## 7.1 완성도 항목

| 항목 | 비중 예시 |
|---|---:|
| 필수 인증 완료 | 25% |
| 사진 등록 | 15% |
| 기본 정보 입력 | 15% |
| 관계 목적/문체 응답 | 20% |
| 나의 프롤로그 작성 | 15% |
| 라이프스타일/취향 키워드 | 10% |

## 7.2 표시 원칙

- 완성도를 점수 경쟁처럼 보이게 하지 않는다.
- “더 좋은 추천을 위한 준비도”로 설명한다.
- 낮은 완성도 사용자를 부끄럽게 만들지 않는다.

### 예시 문구

> 프로필을 조금 더 채우면, 관계 성향이 맞는 사람에게 추천될 가능성이 높아져요.

# 8. 프로필 공개 범위

## 8.1 공개 범위 원칙

1. 인증 원본은 공개하지 않는다.
2. 인증 결과는 배지로 표시한다.
3. 민감 정보는 사용자가 공개 여부를 조정할 수 있어야 한다.
4. 추천 카드와 상세 프로필의 정보 깊이를 분리한다.
5. 차단/신고 이후에는 프로필 노출을 제한한다.

## 8.2 공개 수준 후보

| 정보 | 추천 카드 | 상세 프로필 | 비고 |
|---|---|---|---|
| 대표 사진 | 공개 | 공개 | 추후 단계적 공개 검토 |
| 나이 | 공개 | 공개 | 생년월일 비공개 |
| 지역 | 요약 공개 | 시/구 단위 공개 | 상세 주소 비공개 |
| 직업군 | 공개 | 공개 | 회사명은 선택 |
| 인증 배지 | 공개 | 공개 | 원본 비공개 |
| 관계 목적 | 요약 공개 | 상세 일부 공개 | 민감 문항 제외 가능 |
| 나의 프롤로그 | 일부 공개 | 전체 공개 | 카드에는 요약 |
| 이야기의 목차 | 일부 공개 | 전체 공개 | 작성자가 공개 범위 선택 가능 |
| 연락 빈도/갈등 방식 | 요약 | 상세 | 매칭 리포트에도 활용 |

# 9. 추천 카드와의 연결

프로필 정보는 추천 카드에서 다음 방식으로 요약된다.

## 9.1 추천 카드에 노출할 정보

- 대표 사진
- 나이
- 지역
- 직업군
- 핵심 인증 배지
- 관계 목적 요약
- 라이프스타일 키워드 2~3개
- 나의 프롤로그 첫 문장
- 큐레이터의 메모

## 9.2 상세 프로필로 넘길 정보

- 추가 사진
- 나의 프롤로그 전체
- 이야기의 목차
- 관계의 문체
- 상세 라이프스타일
- 신뢰 배지 상세
- 첫 대화 키워드

---

# 10. 매칭 리포트와의 연결

프로필 항목은 매칭 리포트의 근거가 된다.

| 프로필 항목 | 리포트 활용 |
|---|---|
| 관계 목적 | 추천 적합도 설명 |
| 관계 시작 속도 | 잘 맞는 지점/차이점 |
| 연락 빈도 | 확인하면 좋은 차이점 |
| 주말 생활 | 공통 라이프스타일 |
| 취향 키워드 | 첫 대화 주제 |
| 갈등 해결 방식 | 관계 성향 설명 |
| 인증 배지 | 신뢰 기반 추천 설명 |
| 프로필 완성도 | 추천 신뢰도 보조 |

# 11. 프로필 검수 정책 방향

## 11.1 제한해야 할 내용

- 외부 연락처 노출
- 금전 요구 또는 조건 거래 암시
- 성적 표현
- 혐오/차별 표현
- 욕설 또는 공격적 표현
- 허위 정보
- 타인 사칭
- 불법 행위 암시
- 결혼중개나 성매매로 오해될 수 있는 표현

## 11.2 검수 방식 후보

- 자동 필터링
- 사용자 신고 기반 검수
- 고위험 문구 수동 검토
- 사진 검수 API 또는 수동 검토

## 11.3 반려 안내 원칙

- 사용자를 비난하지 않는다.
- 어떤 항목이 문제인지 구체적으로 안내한다.
- 수정 후 재제출 가능성을 제공한다.

예시:

> 프로필에 외부 연락처가 포함되어 있어 공개할 수 없어요. 연락처를 제외하고 다시 작성해주세요.
