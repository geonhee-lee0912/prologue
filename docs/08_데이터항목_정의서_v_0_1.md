# 1. 문서 목적

이 문서는 프롤로그 MVP에서 수집, 저장, 활용해야 할 주요 데이터 항목을 정의한다.

목적은 사용자, 인증, 프로필, 추천, 매칭, 대화, 신고, 결제 데이터를 일관된 구조로 관리하는 것이다.

이 문서는 다음 작업의 기준 문서로 사용된다.

1. 데이터베이스 설계
2. API 설계
3. 개인정보/공개 범위 정책 검토
4. 추천 알고리즘 설계
5. 관리자 도구 설계
6. QA 데이터 검증
7. 로그/지표 설계

---

# 2. 데이터 설계 원칙

1. 인증 원본은 최소한으로 저장한다.
2. 인증 결과는 상태값과 배지 중심으로 관리한다.
3. 민감한 정보는 공개 데이터와 내부 운영 데이터를 분리한다.
4. 추천 알고리즘에 필요한 데이터와 사용자에게 공개되는 데이터를 구분한다.
5. 신고/제재, 차단, 대화 안전을 위한 데이터는 운영 목적과 보관 기간을 명확히 둔다.
6. 탈퇴, 삭제 요청, 계정 정지 상태를 데이터 구조에서 처리할 수 있어야 한다.
7. 사용자에게 공개되는 정보는 공개 범위 설정과 연결되어야 한다.
8. MVP 단계에서는 과도한 민감정보 수집을 피한다.

# 4. 공통 필드 규칙

대부분의 엔티티는 다음 공통 필드를 가질 수 있다.

| 필드명 | 타입 | 설명 |
|---|---|---|
| created_at | datetime | 생성일 |
| updated_at | datetime | 수정일 |
| deleted_at | datetime | 삭제일 또는 소프트 삭제일 |
| status | enum | 상태값 |

## 4.1 ID 규칙

- 모든 주요 엔티티는 고유 ID를 가진다.
- 외부에 노출되는 ID와 내부 DB ID는 분리할 수 있다.
- 사용자 식별자는 `user_id`를 기준으로 한다.

## 4.2 상태값 규칙

상태값은 가능하면 다음 패턴을 따른다.

- `active`
- `pending`
- `approved`
- `rejected`
- `suspended`
- `withdrawn`
- `expired`
- `blocked`
