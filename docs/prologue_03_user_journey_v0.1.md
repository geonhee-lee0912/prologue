# 1. 문서 목적

이 문서는 프롤로그 사용자가 서비스를 처음 인지한 순간부터 가입, 인증, 프로필 작성, 추천 확인, 관심/거절, 상호 관심, 대화, 재방문까지 이어지는 전체 사용자 여정을 정의하기 위한 문서다.

프롤로그는 단순히 가입 후 프로필을 넘기는 소개팅 앱이 아니다. 사용자가 “이 서비스는 기존 소개팅 앱보다 신뢰할 수 있고, 결정사보다 부담이 낮다”고 느껴야 하며, 그 인식은 랜딩페이지의 카피만으로 만들어지지 않는다. 온보딩, 인증 안내, 프로필 작성 방식, 추천 카드, 매칭 리포트, 대화 시작까지 이어지는 전체 흐름이 같은 메시지를 전달해야 한다.

이 문서는 다음 의사결정의 기준으로 활용한다.

1. MVP에서 반드시 구현해야 하는 핵심 흐름을 정의한다.
2. 사용자가 어떤 순간에 기대, 부담, 불안을 느끼는지 파악한다.
3. 온보딩, 인증, 프로필, 추천, 대화 화면의 역할을 분리한다.
4. 이탈 위험이 높은 지점을 미리 정의하고 완화 전략을 세운다.
5. 제품 지표와 사용자 리서치 항목을 설계한다.
6. 이후 IA, 화면 목록, 기능요구사항서로 전환할 수 있는 기준을 만든다.

# 2. 사용자 여정 한 줄 정의

**프롤로그의 사용자 여정은 사용자가 가볍지 않은 만남에 대한 기대를 갖고 들어와, 신뢰를 위한 인증과 이야기형 프로필을 작성한 뒤, 제한된 추천과 매칭 리포트를 통해 납득 가능한 상대를 만나고 대화를 시작하는 흐름이다.**

핵심 흐름은 다음과 같다.

> 인지 → 사전 신청/가입 → 인증 → 프로필 작성 → 관계 목적 확인 → 추천 준비 → 추천 카드 → 매칭 리포트 → 관심/거절 → 상호 관심 → 대화 → 피드백/재방문

---

# 3. 사용자 여정 설계 원칙

## 3.1 쉽지만 가볍지 않게

앱처럼 쉽게 시작할 수 있어야 하지만, 기존 소개팅 앱처럼 가볍게 소비되는 느낌을 주어서는 안 된다. 가입과 인증은 가능한 한 간결하게 설계하되, 왜 필요한 절차인지 명확히 설명한다.

## 3.2 신뢰는 초반에 만들고, 부담은 단계적으로 낮춘다

프롤로그는 인증 기반 서비스이므로 초반에 신뢰 장치를 보여줘야 한다. 다만 모든 인증을 한 번에 요구하면 이탈이 커질 수 있다. 필수 인증과 선택 인증을 분리하고, 사용자가 각 인증의 목적과 공개 범위를 이해하게 해야 한다.

## 3.3 추천 전까지 기대를 쌓는다

사용자가 인증과 프로필 작성을 완료하는 이유는 결국 좋은 추천을 받기 위해서다. 온보딩 과정에서는 “이 과정을 마치면 더 납득 가능한 추천을 받을 수 있다”는 기대를 유지해야 한다.

## 3.4 추천은 설명되어야 한다

프롤로그의 추천은 단순 카드 노출이 아니다. 추천 카드는 상대를 처음 이해하는 화면이고, 매칭 리포트는 추천의 이유를 납득시키는 핵심 화면이다.

## 3.5 안전감은 전 여정에 걸쳐 제공한다

안전은 신고/차단 화면에만 존재하는 기능이 아니다. 정보 공개 범위, 인증 안내, 대화 시작, 거절 사유 입력, 신고/차단까지 모든 단계에서 사용자가 통제감을 느껴야 한다.

# 4. 전체 여정 요약

| 단계 | 사용자 마음 | 핵심 UX | 주요 이탈 리스크 | 핵심 지표 |
|---|---|---|---|---|
| 1. 인지 | “이게 어떤 서비스지?” | 포지셔닝과 문제 제기 | 소개팅 앱인지 이해 못함 | 랜딩 체류, CTA 클릭 |
| 2. 사전 신청/가입 | “한번 해볼까?” | 간단한 가입 시작 | 인증 부담 예상 | 가입 시작률 |
| 3. 필수 인증 | “이 정도는 해야 하나?” | 본인/얼굴/나이 인증 | 귀찮음, 개인정보 불안 | 인증 완료율 |
| 4. 관계 목적 확인 | “나와 맞는 사람을 만날 수 있을까?” | 관계 목적/속도/연락 설문 | 질문이 무겁게 느껴짐 | 설문 완료율 |
| 5. 프로필 작성 | “내 이야기를 어떻게 써야 하지?” | 이야기형 프로필 입력 | 작성 부담 | 프로필 완료율 |
| 6. 추천 준비 | “언제 추천받지?” | 추천 준비 상태 안내 | 대기 불안, 기대 하락 | 추천 대기 이탈률 |
| 7. 추천 카드 | “이 사람은 어떤 사람이지?” | 오늘의 프롤로그 | 정보 부족, 호감 낮음 | 카드 열람률 |
| 8. 매칭 리포트 | “왜 추천됐지?” | 추천 이유/차이점/대화 주제 | 설명이 뻔함 | 리포트 열람률 |
| 9. 관심/거절 | “더 알아볼까?” | 관심 남기기/넘기기 | 행동 부담, 선택 피로 | 관심률, 거절 사유 입력률 |
| 10. 상호 관심 | “대화해도 괜찮을까?” | 매칭 안내, 대화 시작 | 첫 대화 부담 | 대화 시작률 |
| 11. 대화 | “어떻게 시작하지?” | 첫 대화 주제, 신고/차단 | 대화 단절, 비매너 | 첫 응답률, 신고율 |
| 12. 재방문 | “또 볼 만한 사람이 있을까?” | 다음 추천, 피드백 반영 | 추천 품질 불만 | 재방문율, 잔존율 |

# 5. 단계별 상세 여정

## 5.1 1단계: 서비스 인지

### 사용자 상황

사용자는 SNS, 지인 공유, 검색, 광고, 커뮤니티 글, 앱스토어 등을 통해 프롤로그를 처음 접한다.

### 사용자 질문

- 이게 소개팅 앱인가?
- 기존 소개팅 앱과 뭐가 다른가?
- 결정사 같은 건가?
- 인증은 얼마나 해야 하나?
- 나에게 맞는 서비스인가?

### 제품/콘텐츠 역할

- 첫 화면에서 “가볍지 않은 만남을 위한 인증 소개팅”을 명확히 전달한다.
- 책방/프롤로그 콘셉트는 감성 보조로 사용하되, 카테고리 인지를 방해하지 않는다.
- 문제 제기는 기존 소개팅 앱의 피로감과 결정사의 부담 사이에 둔다.

### 핵심 메시지

> 소개팅 앱은 많지만, 믿고 만날 사람은 적다고 느껴질 때.

### 성공 기준

- 사용자가 프롤로그를 소개팅 앱으로 이해한다.
- 기존 소개팅 앱보다 신뢰감 있는 서비스로 인식한다.
- 결정사처럼 무겁지는 않다고 느낀다.

---

## 5.2 2단계: 사전 신청 또는 가입 시작

### 사용자 상황

사용자는 서비스가 궁금해져 사전 신청 또는 가입을 시작한다.

### 사용자 질문

- 얼마나 걸릴까?
- 어떤 정보를 입력해야 할까?
- 내 정보가 어디까지 공개될까?
- 가입하면 바로 추천을 받을 수 있을까?

### 제품 역할

- 가입 전 전체 과정을 간단히 안내한다.
- 필수 정보와 선택 정보를 분리한다.
- 인증과 프로필 작성의 목적을 설명한다.
- “추천을 위한 준비 과정”이라는 기대를 만든다.

### 권장 화면 요소

- 예상 소요 시간
- 필수 단계 안내
- 개인정보/공개 범위 안내 링크
- 사전 신청 또는 가입 CTA

### 예시 문구

> 좋은 추천을 위해 몇 가지 확인이 필요해요.  
> 입력한 정보는 공개 범위에 따라 단계적으로 보여집니다.

---

## 5.3 3단계: 필수 인증

### 사용자 상황

사용자는 본인 인증, 얼굴 인증, 나이 확인 등 기본 신뢰 절차를 진행한다.

### 사용자 질문

- 왜 인증해야 하지?
- 내 신분증이나 사진이 공개되나?
- 인증에 실패하면 어떻게 되나?
- 이 과정이 너무 결정사처럼 무거운 건 아닐까?

### 제품 역할

- 인증 목적을 “심사”가 아니라 “허위 프로필 방지”로 설명한다.
- 원본 정보와 공개 정보가 다르다는 점을 명확히 한다.
- 인증 완료 후 받는 신뢰 배지를 보여준다.
- 실패 시 재시도와 도움말을 제공한다.

### 필수 인증 후보

- 본인 인증
- 얼굴 인증
- 나이 확인
- 싱글 상태 서약
- 매너 서약

### 예시 문구

> 얼굴 인증은 도용 사진을 줄이기 위한 과정입니다.  
> 인증 결과는 배지로 표시되며, 원본 이미지는 상대에게 공개되지 않아요.

---

## 5.4 4단계: 관계 목적 확인

### 사용자 상황

사용자는 어떤 만남을 원하는지, 관계를 어떤 속도로 시작하고 싶은지 응답한다.

### 사용자 질문

- 너무 결혼 중심으로 묻는 건 아닐까?
- 내 답변이 상대에게 그대로 공개될까?
- 이 질문이 추천에 어떻게 쓰일까?

### 제품 역할

- 관계 목적을 부담스럽지 않게 묻는다.
- 결혼을 강요하거나 전제하지 않는다.
- 답변이 추천 품질에 활용된다는 점을 설명한다.
- 일부 답변은 프로필에 요약 표시될 수 있음을 안내한다.

### 질문 영역

- 지금 원하는 만남의 목적
- 좋은 사람을 만났을 때 생각 가능한 관계 범위
- 관계 시작 속도
- 연락 빈도
- 대화 스타일
- 갈등 해결 방식

### 예시 문구

> 관계를 시작하는 속도와 온도가 비슷한 사람을 추천하기 위한 질문이에요.

---

## 5.5 5단계: 이야기형 프로필 작성

### 사용자 상황

사용자는 자신의 사진, 기본 정보, 자기소개, 라이프스타일, 관계 성향을 입력한다.

### 사용자 질문

- 무엇을 얼마나 써야 하지?
- 너무 긴 글을 써야 하나?
- 내가 어떻게 보일까?
- 사진과 기본 정보 말고 뭘 더 보여줄 수 있을까?

### 제품 역할

- 긴 글보다 짧은 문장형 문답으로 시작한다.
- 예시 문구와 선택형 답변을 제공한다.
- 프로필 완성도가 추천 품질에 영향을 준다는 점을 알려준다.
- 사진 중심이 아니라 이야기와 관계 성향을 함께 보여준다.

### 핵심 구성

- 사진
- 기본 정보
- 나의 프롤로그
- 이야기의 목차
- 관계의 문체
- 라이프스타일 키워드
- 신뢰 배지

### 예시 문구

> 완벽한 소개보다, 당신다운 문장이 더 좋은 추천을 만듭니다.

---

## 5.6 6단계: 추천 준비 상태

### 사용자 상황

사용자는 인증과 프로필을 완료한 뒤 추천을 기다린다.

### 사용자 질문

- 이제 무엇을 하면 되지?
- 추천은 언제 오지?
- 내 프로필이 충분한가?
- 더 인증하면 추천이 좋아지나?

### 제품 역할

- 추천 준비 완료 상태를 명확히 보여준다.
- 부족한 항목이 있으면 개선 가이드를 제공한다.
- 선택 인증의 이점을 설명한다.
- 첫 추천을 기다리는 불안을 줄인다.

### 화면 요소

- 추천 준비 완료 메시지
- 프로필 완성도
- 선택 인증 안내
- 다음 추천 예상 시점
- 프로필 미리보기

### 예시 문구

> 추천 준비가 완료되었어요.  
> 프롤로그 큐레이터가 관계 성향과 프로필 정보를 바탕으로 어울릴 만한 사람을 정리할게요.

---

## 5.7 7단계: 추천 카드 확인

### 사용자 상황

사용자는 오늘의 추천을 확인한다.

### 사용자 질문

- 이 사람은 어떤 사람이지?
- 믿을 수 있는 정보가 있나?
- 나와 어떤 점이 맞을까?
- 더 볼 만한 이유가 있나?

### 제품 역할

- 첫 화면에서 상대의 핵심 정보를 빠르게 이해하게 한다.
- 인증 배지를 명확히 보여준다.
- 외모와 조건뿐 아니라 관계 성향과 이야기 요약을 보여준다.
- 큐레이터의 메모로 추천 이유를 간단히 제공한다.

### 추천 카드 요소

- 대표 사진
- 나이/지역/직업군
- 인증 배지
- 관계 목적 요약
- 라이프스타일 키워드
- 큐레이터의 메모
- 관심 남기기/넘기기

### 예시 문구

> 두 분은 관계를 천천히 시작하는 편이고, 주말을 보내는 방식이 비슷해요.

---

## 5.8 8단계: 매칭 리포트 확인

### 사용자 상황

사용자는 추천 카드를 보고 더 자세한 이유를 확인한다.

### 사용자 질문

- 왜 이 사람이 나에게 추천됐지?
- 어떤 점이 비슷하지?
- 어떤 차이를 미리 알아야 하지?
- 대화를 시작하면 무엇을 물어보면 좋을까?

### 제품 역할

- 추천의 근거를 납득 가능하게 보여준다.
- 잘 맞는 지점과 확인할 차이점을 균형 있게 제공한다.
- 단정적 표현보다 가능성 중심 표현을 사용한다.
- 첫 대화 주제를 제안한다.

### 리포트 요소

- 추천 요약
- 잘 맞는 지점
- 비슷한 관계 성향
- 공통 라이프스타일
- 확인하면 좋은 차이점
- 첫 대화 추천 주제
- 데이터 기반 안내

### 예시 문구

> 이 추천은 관계 목적, 라이프스타일 응답, 프로필 완성도를 바탕으로 생성되었어요.

---

## 5.9 9단계: 관심 또는 거절

### 사용자 상황

사용자는 상대에게 관심을 남기거나 추천을 넘긴다.

### 사용자 질문

- 관심을 보내면 상대가 무엇을 보게 되지?
- 거절하면 상대에게 알려지나?
- 넘기는 이유를 꼭 알려야 하나?

### 제품 역할

- 관심 표현의 의미를 명확히 설명한다.
- 거절은 상대에게 공개되지 않는다는 점을 안내한다.
- 거절 사유는 가장 가까운 이유 하나만 선택하도록 한다.
- 피드백이 추천 개선에 쓰인다는 점을 설명한다.

### 예시 문구

> 더 나은 추천을 위해 가장 가까운 이유 하나만 알려주세요.  
> 이 내용은 상대에게 공개되지 않아요.

---

## 5.10 10단계: 상호 관심과 대화 시작

### 사용자 상황

서로 관심을 남긴 경우 대화가 열린다.

### 사용자 질문

- 어떻게 시작하지?
- 첫 메시지를 뭐라고 보내야 하지?
- 상대가 부담스러워하지 않을까?
- 문제가 생기면 차단할 수 있나?

### 제품 역할

- 상호 관심을 브랜드 카피로 기분 좋게 안내한다.
- 첫 대화 주제를 제공한다.
- 신고/차단 버튼을 명확히 제공한다.
- 매너 있는 대화를 유도한다.

### 예시 문구

> 두 사람의 첫문장이 이어졌어요.  
> 이제 대화를 시작할 수 있습니다.

### 첫 대화 주제 예시

- 두 사람이 공통으로 선택한 취향 키워드
- 주말을 보내는 방식
- 관계를 시작할 때 중요하게 생각하는 것
- 최근 즐겁게 하고 있는 일

---

## 5.11 11단계: 대화 경험

### 사용자 상황

사용자는 상대와 실제 대화를 시작한다.

### 사용자 질문

- 대화가 자연스럽게 이어질까?
- 상대가 무례하면 어떻게 하지?
- 대화가 끊기면 다음 추천을 받을 수 있나?

### 제품 역할

- 대화 시작 부담을 낮춘다.
- 신고/차단/나가기 기능을 명확히 제공한다.
- 비매너 행동을 예방하기 위한 안내를 제공한다.
- 대화 품질을 높일 수 있는 가벼운 가이드를 제공한다.

### MVP 포함 범위

- 기본 채팅
- 신고/차단
- 대화방 나가기
- 첫 대화 주제 표시

### MVP 제외 범위

- AI 자동 답변
- 복잡한 관계 코칭
- 음성/영상 통화
- 위치 기반 만남 제안

---

## 5.12 12단계: 피드백과 재방문

### 사용자 상황

사용자는 추천을 소비한 뒤 다음 추천을 기다리거나 프로필을 개선한다.

### 사용자 질문

- 다음 추천은 언제 오지?
- 내 피드백이 반영될까?
- 프로필을 더 채우면 추천이 좋아질까?
- 이 앱을 계속 쓸 이유가 있을까?

### 제품 역할

- 다음 추천 예상 시점을 알려준다.
- 거절 사유와 관심 행동이 추천 개선에 반영된다는 느낌을 준다.
- 프로필 개선 가이드를 제공한다.
- 선택 인증의 이점을 안내한다.

### 예시 문구

> 오늘 남긴 피드백을 바탕으로 다음 추천을 더 정교하게 준비할게요.

# 6. 세그먼트별 여정 차이

## 6.1 26~30세 진지한 연애 시작층

이 세그먼트는 인증과 관계 목적 질문이 너무 무겁게 느껴질 수 있다.

설계 포인트:

- 결혼보다 “가볍지 않은 연애”를 강조한다.
- 인증은 기본 신뢰 장치로 설명한다.
- 프로필 작성은 감성적이고 부담 없게 시작한다.
- 선택 인증은 나중에 유도한다.

## 6.2 30~35세 관계 안정 추구층

이 세그먼트는 프롤로그의 핵심 사용자다.

설계 포인트:

- 시간 낭비를 줄이는 추천을 강조한다.
- 인증 배지와 추천 이유를 명확히 보여준다.
- 매칭 리포트의 납득도를 높인다.
- 유료화나 고신뢰 풀에 대한 수용 가능성을 검증한다.

## 6.3 35~39세 관계 명확성 추구층

이 세그먼트는 신뢰와 효율을 중요하게 본다.

설계 포인트:

- 추천 근거를 실질적으로 보여준다.
- 직업/재직 인증, 싱글 상태 확인, 생활권 정보를 명확히 다룬다.
- 책방 메타포는 과하지 않게 사용한다.
- 매칭 리포트는 감성보다 정보 중심으로 구성한다.

# 7. 여성/남성 사용자 관점의 여정 차이

## 7.1 여성 사용자

여성 사용자는 전 여정에서 안전감과 통제감을 중요하게 느낄 수 있다.

중요 지점:

- 가입 전 정보 공개 범위 안내
- 본인/얼굴 인증의 신뢰 효과
- 사진 노출 범위
- 관심을 받은 뒤 상대 정보 확인
- 대화 전 신고/차단 가능 여부
- 불편한 대화 대응

핵심 설계 원칙:

> 여성 사용자가 “내 정보와 대화 흐름을 내가 통제할 수 있다”고 느끼게 해야 한다.

## 7.2 남성 사용자

남성 사용자는 추천 효율과 인증 보상을 중요하게 느낄 수 있다.

중요 지점:

- 인증 완료 시 추천 신뢰도 향상 안내
- 프로필 완성도에 따른 추천 기회
- 무의미한 스와이프가 줄어드는 가치
- 관심을 보낼 때 상대에게 어떻게 보이는지
- 대화 시작 가능성을 높이는 첫 대화 주제

핵심 설계 원칙:

> 남성 사용자가 “성의 있게 인증하고 프로필을 채우면 더 좋은 추천 기회를 얻을 수 있다”고 느끼게 해야 한다.

# 8. MVP 핵심 여정

MVP에서 반드시 검증해야 하는 핵심 여정은 다음이다.

5. 싱글 상태/매너 서약 화면

10. 추천 카드 화면
11. 매칭 리포트 화면
12. 관심/거절 화면
13. 거절 사유 입력 화면
14. 상호 관심 완료 화면
15. 채팅 화면
16. 신고/차단 화면

## 8.2 MVP 제외 화면

- 커뮤니티
- 피드
