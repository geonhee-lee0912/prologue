# 프롤로그 서비스 콘셉트 기획서 v1.1

## 0. 문서 목적

이 문서는 인증 기반 진지한 소개팅 앱 **프롤로그**의 초기 서비스 콘셉트를 정리한 기획 문서다. v1.1에서는 BM 방향, 추가 추천권, 내부 대화방 운영 정책, 타로/사주 궁합 부가 콘텐츠 검토 방향을 반영한다.

현재 단계의 목적은 개발 상세 요구사항을 정의하는 것이 아니라, 서비스가 어떤 문제를 해결하고, 어떤 사용자에게 어떤 경험을 제공할지 명확히 정리하는 데 있다. 이후 바이브코딩을 통해 MVP 또는 베타 앱을 개발할 때, 이 문서는 제품 방향성과 우선순위를 판단하는 기준점으로 활용한다.

# 1. 서비스 개요

## 1.1 서비스명

**프롤로그**

## 1.2 부제

**가볍지 않은 만남을 위한 인증 소개팅**

## 1.3 대표 카피

**좋은 만남은, 좋은 첫문장에서 시작됩니다.**

## 1.4 한 줄 정의

**프롤로그는 가볍지 않은 만남을 원하는 사람들이 인증된 프로필과 관계 성향을 바탕으로 서로의 이야기를 시작하는 소개팅 앱이다.**

## 1.5 핵심 콘셉트

프롤로그는 기존 소개팅 앱의 접근성은 유지하되, 사용자가 느끼는 불신과 피로감을 줄이기 위해 인증, 관계 목적 확인, 큐레이션 추천, 매칭 리포트를 결합한 서비스다.

서비스는 결혼중개 또는 결정사 대체 서비스를 전면에 내세우지 않는다. 대신 **신뢰할 수 있는 사람들과 진지한 관계를 시작하는 소개팅 앱**으로 포지셔닝한다.

# 2. 서비스가 해결하려는 문제

## 2.1 기존 소개팅 앱의 문제

기존 소개팅 앱은 접근성이 높고 사용이 쉽지만, 사용자 입장에서는 다음과 같은 문제가 있다.

1. 상대가 진지한 만남을 원하는지 알기 어렵다.
2. 프로필 정보가 실제와 일치하는지 신뢰하기 어렵다.
3. 기혼자, 애인이 있는 사람, 목적이 다른 사람이 섞여 있을 수 있다.
4. 사진과 외모 중심으로 빠르게 판단하게 된다.
5. 대화는 많이 생기지만 실제 좋은 만남으로 이어지지 않는 경우가 많다.
6. 잠수, 노쇼, 무례한 대화, 허위 정보 등 비매너 경험이 피로감을 만든다.
7. 상호 매칭 이후에도 대화가 방치되거나, 너무 이른 연락처 교환 요구가 부담이 될 수 있다.

## 2.2 기존 결혼정보회사/결정사의 문제

결정사는 신뢰도와 진지함은 있지만, 초기 사용자가 느끼는 진입장벽이 높다.

1. 가격이 높고 부담스럽다.
2. 상담과 가입 과정이 무겁게 느껴진다.
3. 사무실 방문, 계약, 환불 등 절차가 복잡하다.
4. 아직 결정사를 이용할 정도는 아니라는 심리적 저항이 있다.
5. 너무 조건 중심으로 사람을 판단하는 느낌이 있다.

## 2.3 프롤로그의 문제 정의

**소개팅 앱은 너무 가볍고, 결정사는 너무 무겁다.**

프롤로그는 이 사이에서 다음 문제를 해결한다.

> 사용자는 앱처럼 쉽게 시작하고 싶지만, 아무나 만나는 가벼운 경험은 원하지 않는다.\
> 프롤로그는 인증된 정보와 관계 성향을 바탕으로, 신뢰할 수 있는 진지한 만남을 제공한다.

# 3. 포지셔닝

## 3.1 서비스 정체성

**인증 기반 진지한 소개팅 앱**

## 3.2 우리가 아닌 것

프롤로그는 다음과 같은 서비스가 아니다.

- 결혼중개 서비스
- 성혼을 약속하는 서비스
- 결정사 대체 서비스
- 무한 스와이프 중심의 데이팅 앱
- 외모와 조건만으로 사람을 소비하는 앱
- 가벼운 만남을 장려하는 앱

## 3.3 우리가 지향하는 것

프롤로그는 다음과 같은 서비스다.

- 인증된 사람을 만날 수 있는 소개팅 앱
- 관계 목적이 비교적 분명한 사람들이 모이는 앱
- 사진만이 아니라 한 사람의 이야기와 성향을 함께 보는 앱
- 무한 탐색이 아니라 제한된 큐레이션 추천을 제공하는 앱
- 사람을 조건표가 아니라 하나의 이야기로 소개하는 앱

## 3.4 핵심 포지션 문장

**프롤로그는 더 많은 사람을 보여주는 앱이 아니라, 덜 의심해도 되는 사람을 신중하게 소개하는 앱이다.**

# 4. 브랜드 콘셉트

## 4.1 브랜드 메타포

프롤로그의 브랜드 메타포는 **책방/서점**이다.

사람의 프로필을 단순한 조건표가 아니라, 한 권의 이야기처럼 보여준다. 앱은 책방 주인이 독자에게 책을 추천하듯, 사용자에게 어울리는 사람을 추천한다.

## 4.2 브랜드 세계관

**사람은 조건표가 아니라, 하나의 이야기다.**\
**프롤로그는 그 이야기를 신뢰할 수 있게 소개한다.**

## 4.3 책방 콘셉트 적용 방식

책방 콘셉트는 서비스 전체의 감성적 차별화 요소로 가져가되, 사용자가 처음 봤을 때 소개팅 앱이라는 점은 명확하게 인지할 수 있어야 한다.

따라서 외부 커뮤니케이션에서는 다음처럼 명확하게 표현한다.

- 인증 기반 소개팅
- 진지한 만남
- 신뢰할 수 있는 프로필
- 관계 성향 기반 추천

앱 내부 경험에서는 책방 메타포를 적극 활용한다.

| 일반 소개팅 앱 표현 | 프롤로그식 표현           |   |
| ----------- | ------------------ | - |
| 추천 프로필      | 오늘의 프롤로그           |   |
| 프로필         | 이야기 / 프로필북         |   |
| 자기소개        | 나의 프롤로그            |   |
| 상세 문답       | 이야기의 목차            |   |
| 관계 성향       | 관계의 문체             |   |
| 추천 이유       | 큐레이터의 메모           |   |
| 관심 표현       | 책갈피                |   |
| 저장          | 내 서재               |   |
| 상호 매칭       | 첫문장이 이어졌어요         |   |
| 대화 시작       | 첫 페이지 열기 / 대화 시작하기 |   |

## 4.4 주의해야 할 표현

책방 콘셉트는 사람을 존중하며 이해하는 방향으로 사용해야 한다. 사람을 상품이나 물건처럼 느끼게 하는 표현은 피한다.

피해야 할 표현:

- 대출
- 반납
- 품절
- 베스트셀러
- 희귀본
- 중고책
- 구매
- 판매

## 4.5 프롤로그 큐레이터 캐릭터

프롤로그는 추천 알고리즘을 차갑고 기계적인 추천 엔진으로만 보여주지 않고, 책방/서점 콘셉트에 맞춰 **프롤로그 큐레이터** 또는 **책방지기**라는 브랜드 페르소나로 표현한다.

이 캐릭터는 실제 사람이 직접 사용자를 심사하거나 상대를 수동으로 골라주는 존재가 아니다. 대신 알고리즘이 분석한 관계 성향, 프로필 정보, 인증 상태, 라이프스타일 응답을 사용자에게 더 따뜻하고 이해하기 쉬운 언어로 설명해주는 **추천 경험의 해석자** 역할을 한다.

기본 원칙은 다음과 같다.

1. 알고리즘 기반 추천이라는 사실은 투명하게 전달한다.
2. 캐릭터는 추천을 대신 판단하는 존재가 아니라 추천 이유를 설명하는 큐레이터로 사용한다.
3. 캐릭터는 과하게 귀엽거나 유아적인 마스코트가 아니라, 차분하고 신뢰감 있는 에디토리얼 스타일의 안내자로 표현한다.
4. 온보딩, 인증 안내, 추천 카드, 매칭 리포트, 빈 상태 화면 등에서 제한적으로 활용한다.
5. 결제, 신고, 제재, 개인정보 안내처럼 명확성과 신뢰가 중요한 화면에서는 캐릭터 사용을 줄인다.

공식 표현은 **프롤로그 큐레이터**를 우선 사용하고, 앱 내부 감성 문구에서는 **책방지기** 표현을 보조적으로 사용할 수 있다.

예시:

> 프롤로그 큐레이터가 두 사람의 관계 성향을 바탕으로 추천 이유를 정리했어요.  
> 오늘 책방지기가 고른 한 사람의 프롤로그를 읽어보세요.

# 5. 타깃 고객

## 5.1 기본 타깃

**26\~39세 싱글**

단, 단순히 연령만으로 정의하지 않는다. 핵심 타깃은 다음 감정을 가진 사람들이다.

- 소개팅 앱은 써봤지만 너무 가볍다고 느낀 사람
- 상대의 직업, 신원, 관계 목적을 신뢰하기 어려웠던 사람
- 사진만 보고 넘기는 방식에 피로감을 느낀 사람
- 가벼운 대화보다 진지한 관계 가능성을 원하지만, 결정사는 아직 부담스러운 사람
- 허위 프로필, 기혼자, 비매너, 노쇼 등에 피로감을 느낀 사람

## 5.2 내부 세그먼트

| 세그먼트       | 연령      | 핵심 니즈                  | 주요 메시지                  |
| ---------- | ------- | ---------------------- | ----------------------- |
| 진지한 연애 시작층 | 26\~30세 | 가볍지 않은 연애, 검증된 상대      | 처음부터 목적이 다른 사람을 만나세요    |
| 관계 안정 추구층  | 30\~35세 | 시간 낭비 없는 만남, 신뢰 가능한 상대 | 신뢰할 수 있는 사람을 이야기로 만나세요  |
| 관계 명확성 추구층 | 35\~39세 | 목적과 조건의 명확성, 효율적인 만남   | 확인된 프로필과 관계 성향으로 추천받으세요 |

## 5.3 성별 관점

프롤로그는 남녀 모두에게 필요한 서비스지만, 초기 신뢰 형성에서는 여성 사용자의 안전감과 통제감을 특히 중요하게 봐야 한다.

여성 사용자가 중요하게 느낄 가치:

- 상대가 실제 싱글인지
- 허위 프로필이 아닌지
- 관계 목적이 가볍지 않은지
- 사진과 정보가 무분별하게 노출되지 않는지
- 신고와 제재가 제대로 작동하는지

남성 사용자가 중요하게 느낄 가치:

- 진지하게 만날 의향이 있는 상대를 만날 수 있는지
- 지나친 경쟁이나 무의미한 스와이프를 줄일 수 있는지
- 추천 이유가 납득 가능한지
- 실제 대화와 만남으로 이어질 가능성이 높은지

# 6. 핵심 가치 제안

## 6.1 신뢰

본인, 얼굴, 관계 목적, 직업/재직 등 주요 정보를 인증하거나 확인함으로써 허위 프로필 가능성을 낮춘다.

## 6.2 진정성

가벼운 만남이 아니라 진지한 관계를 원하는 사용자가 모이도록 관계 목적과 만남 태도를 확인한다.

## 6.3 큐레이션

무한 스와이프가 아니라 제한된 수의 추천을 제공한다. 사용자는 많은 사람을 빠르게 넘기는 대신, 더 신중하게 한 사람을 읽어본다.

## 6.4 이야기형 프로필

사람을 조건표로만 보여주지 않는다. 자기소개, 관계 성향, 라이프스타일, 대화 스타일, 가치관 등을 함께 보여준다.

## 6.5 안전감

허위 정보, 비매너, 노쇼, 기혼자 리스크를 줄이고, 신고/차단/제재 구조를 통해 서비스 신뢰도를 유지한다.

# 7. 인증 정책 v1

프롤로그의 인증은 결혼 조건 검증이 아니라, **허위 프로필과 불성실한 만남을 줄이기 위한 신뢰 장치**다.

## 7.1 인증 원칙

1. 인증은 강하게, 공개는 부드럽게 한다.
2. 원본 정보보다 인증 결과와 신뢰 배지를 중심으로 보여준다.
3. 민감한 정보는 사용자가 공개 범위를 통제할 수 있어야 한다.
4. 인증을 많이 한 사용자가 더 좋은 추천 경험을 얻도록 설계한다.
5. 모든 인증은 서비스 신뢰와 안전을 위한 목적임을 명확히 설명한다.

## 7.2 필수 인증

매칭을 받기 위해 필요한 기본 조건이다.

| 인증 항목    | 목적              |
| -------- | --------------- |
| 본인 인증    | 가짜 계정 방지        |
| 얼굴 인증    | 도용 사진 방지        |
| 나이 인증    | 타깃 연령 확인        |
| 관계 목적 설문 | 가벼운 목적 필터링      |
| 매너 서약    | 노쇼, 잠수, 비매너 방지  |
| 싱글 상태 서약 | 기혼자 또는 연인 있음 방지 |

## 7.3 강력 권장 인증

추천 품질과 노출 우선순위에 영향을 주는 인증이다.

| 인증 항목    | 목적           |
| -------- | ------------ |
| 직업/재직 인증 | 허위 직업 방지     |
| 거주 지역 인증 | 실제 만남 가능성 확인 |
| 학력 인증    | 선택적 신뢰 강화    |

## 7.4 선택/프리미엄 인증

초기에는 전면에 두지 않되, 향후 고신뢰 풀 또는 프리미엄 기능과 연결할 수 있다.

| 인증 항목     | 운영 방향              |
| --------- | ------------------ |
| 소득 구간 인증  | 선택 배지 또는 프리미엄 풀 조건 |
| 경제 안정성 인증 | 정확한 금액보다 인증 여부 중심  |
| 싱글 상태 증빙  | 고신뢰 풀에서 선택 또는 필수   |

## 7.5 표현 방식

기존 표현을 더 부드러운 서비스 표현으로 바꾼다.

| 직접적 표현  | 프롤로그식 표현  |
| ------- | --------- |
| 혼인상태 인증 | 싱글 상태 확인  |
| 소득 인증   | 경제 안정성 인증 |
| 결혼 의향   | 관계 목적     |
| 결혼관     | 장기 관계 가치관 |
| 배우자 조건  | 선호 관계 조건  |
| 결정사급 검증 | 신뢰 프로필 인증 |

# 8. 관계 목적 설문 방향

## 8.1 설문 목적

관계 목적 설문은 사용자가 어떤 속도와 온도로 관계를 시작하고 싶은지 확인하기 위한 장치다.

프롤로그는 결혼을 전면에 내세우지 않지만, 사용자가 장기적인 관계 가능성을 고려하는지는 확인할 수 있다.

## 8.2 질문 방향

질문은 너무 무겁지 않게 시작하되, 사용자의 관계 목적을 충분히 파악할 수 있어야 한다.

예시 질문:

1. 지금 어떤 만남을 원하시나요?

   - 가볍게 알아가는 만남
   - 자연스럽게 시작하는 연애
   - 진지한 연애
   - 장기적인 관계를 염두에 둔 만남

2. 좋은 사람을 만난다면 어느 정도의 관계까지 생각하시나요?

   - 아직은 연애 중심
   - 장기 연애까지 가능
   - 결혼 가능성도 열려 있음
   - 구체적인 미래를 함께 생각하고 싶음

3. 관계를 시작할 때 선호하는 속도는 어떤가요?

   - 천천히 알아가는 편
   - 적당히 자연스럽게 가까워지는 편
   - 확신이 들면 빠르게 가까워지는 편

4. 대화와 연락은 어느 정도를 선호하시나요?

   - 천천히, 부담 없이
   - 하루에 한두 번은 꾸준히
   - 자주 연락하는 편이 좋음

## 8.3 주의점

- “결혼 상대”, “배우자”, “성혼” 같은 표현은 초기 서비스 전면에서 사용하지 않는다.
- 사용자의 장기 관계 의향은 파악하되, 서비스가 결혼중개를 제공하는 것처럼 보이지 않게 한다.
- 설문 결과는 추천 품질 향상과 사용자 경험 개선을 위한 것으로 설명한다.

# 9. 매칭/추천 경험

## 9.0 매칭 플로우 v1.1

프롤로그의 매칭은 추천 기반 쌍방 관심형 매칭으로 설계한다. 사용자가 무한 탐색으로 상대를 찾는 것이 아니라, 프롤로그가 제한된 추천을 제공하고 사용자가 관심 여부를 판단한다.

기본 플로우는 다음과 같다.

> 가입 → 본인/얼굴 인증 → 관계 목적 설문 → 이야기형 프로필 작성 → 추천 후보 생성 → 오늘의 프롤로그 도착 → 추천 카드 열람 → 매칭 리포트 확인 → 관심 보내기 또는 넘기기 → 상대도 관심을 보내면 상호 매칭 → 안전 안내 → 첫 대화 주제 제공 → 내부 대화방 시작

무료 사용자와 유료 사용자는 기본적으로 서로 매칭될 수 있다. 유료화의 차이는 매칭 가능 여부가 아니라 추천 수, 매칭 리포트 깊이, 추가 추천권 혜택, 첫 대화 보조, 고신뢰 풀 접근에서 만든다.

## 9.1 추천 원칙

프롤로그의 추천은 많은 사람을 보여주는 방식이 아니라, 적은 수의 사람을 더 신중하게 소개하는 방식이다.

기본 원칙:

1. 무한 스와이프를 지양한다.
2. 추천 수를 제한한다.
3. 추천 이유를 설명한다.
4. 거절 사유를 학습한다.
5. 외모뿐 아니라 관계 성향과 라이프스타일을 함께 반영한다.
6. 인증 수준과 프로필 완성도를 추천 품질에 반영한다.

## 9.2 추천 빈도

현재 방향은 인증 완료자를 중심으로 추천을 제공하는 것이다. 추천 수와 무료/유료 정책은 추후 BM 설계 단계에서 확정한다.

초기 컨셉상 후보:

- 무료 또는 베타: 제한된 추천
- 인증 완료자: 추가 추천 또는 우선 추천
- 유료 멤버십: 매일 추천 또는 더 정교한 추천 리포트

## 9.2.1 추가 추천권 v1.1

추가 추천권은 기본 추천을 모두 본 뒤, 큐레이션된 추천 카드 1개를 추가로 열람할 수 있는 제한형 유료 상품이다.

추가 추천권은 무한 스와이프나 대량 탐색권이 아니다. 프롤로그의 추천 품질과 신중한 경험을 유지하기 위해 월 구매 한도와 노출 빈도를 제한한다.

운영 원칙:

1. 기본 추천을 모두 본 뒤에만 자연스럽게 노출한다.
2. 추가 추천권 1장은 추가 추천 카드 1개 열람권으로 정의한다.
3. 추가 추천 카드에도 추천 이유와 매칭 리포트 요약을 제공한다.
4. 무제한 구매는 제공하지 않는다.
5. Plus 멤버십에는 월 포함권 또는 할인 혜택을 제공할 수 있다.

## 9.3 추천 카드 구조

추천 카드는 프롤로그의 핵심 화면이다.

### 상단 정보

- 사진
- 나이
- 지역
- 직업군
- 인증 배지

### 중단 정보

- 관계 목적
- 대화 스타일
- 주말 생활
- 연락 빈도
- 흡연/음주
- 취향 키워드

### 하단 정보

- 추천 이유
- 잘 맞는 지점
- 확인하면 좋은 차이점
- 관심 보내기
- 넘기기

## 9.4 큐레이터의 메모 예시

큐레이터의 메모는 프롤로그 큐레이터가 추천 알고리즘의 결과를 사용자가 이해하기 쉬운 문장으로 풀어주는 영역이다. 이 영역은 실제 사람이 수동으로 추천했다는 인상을 주기보다, 관계 성향과 프로필 정보를 바탕으로 추천 이유를 설명하는 방식이어야 한다.

> 두 분은 관계를 천천히 시작하는 편이고, 주말을 보내는 방식이 비슷해요.\
> 둘 다 대화에서 안정감을 중요하게 보는 타입이라, 처음부터 과하게 빠른 관계보다는 자연스럽게 알아가는 만남에 잘 맞을 수 있어요.\
> 다만 연락 빈도에 대한 기대는 조금 다를 수 있어요.

사용 가능한 문구 방향:

- 프롤로그 큐레이터가 관계 성향을 바탕으로 추천 이유를 정리했어요.
- 오늘 어울릴 만한 한 사람의 프롤로그가 도착했어요.
- 이 추천은 프로필, 관계 목적, 라이프스타일 응답을 바탕으로 생성되었어요.

피해야 할 문구 방향:

- 제가 직접 골랐어요.
- 전담 큐레이터가 한 명씩 검토했어요.
- 이 사람은 반드시 잘 맞을 거예요.

## 9.5 거절 사유 수집

사용자가 추천 상대를 넘길 때, 더 나은 추천을 위해 거절 사유를 받는다.

문구 예시:

> 더 나은 추천을 위해 가장 가까운 이유 하나만 알려주세요.\
> 이 내용은 상대에게 공개되지 않아요.

선택지 예시:

- 사진/분위기 호감이 낮아요
- 관계 목적이 달라 보여요
- 거리나 생활권이 맞지 않아요
- 대화 스타일이 맞지 않을 것 같아요
- 직업/생활 패턴이 맞지 않아요
- 취향이나 가치관이 달라 보여요
- 프로필 정보가 부족해요
- 기타

# 10. 프로필 구조

## 10.1 프로필 기본 방향

프롤로그의 프로필은 조건표가 아니라 이야기형 프로필이어야 한다.

사용자는 상대의 외적인 정보뿐 아니라, 어떤 사람인지, 어떤 관계를 원하는지, 어떤 일상을 사는지 이해할 수 있어야 한다.

## 10.2 프로필 구성 예시

### 1. 사진

- 대표 사진
- 일상 사진
- 취향이 드러나는 사진

### 2. 기본 정보

- 나이
- 거주 지역
- 직업군
- 인증 배지

### 3. 나의 프롤로그

자기소개 영역.

예시 질문:

- 나를 가장 잘 설명하는 문장
- 요즘 가장 즐겁게 하는 일
- 내가 편안함을 느끼는 순간
- 좋은 관계에서 중요하게 생각하는 것

### 4. 이야기의 목차

구조화된 문답 영역.

예시 항목:

- 일과 커리어
- 주말과 휴식
- 친구와 가족
- 대화와 연락
- 관계를 시작하는 속도
- 취향과 관심사

### 5. 관계의 문체

관계 성향을 보여주는 영역.

예시 항목:

- 연락 빈도
- 대화 스타일
- 갈등 해결 방식
- 애정 표현 방식
- 독립성과 친밀감의 균형

### 6. 신뢰 배지

인증 결과를 보여주는 영역.

예시:

- 본인 인증 완료
- 얼굴 인증 완료
- 관계 목적 확인 완료
- 직업/재직 인증 완료
- 싱글 상태 서약 완료
- 경제 안정성 인증 선택 완료

# 11. 핵심 화면 방향

## 11.1 홈

홈은 사용자가 오늘 받은 추천을 확인하는 화면이다.

핵심 구성:

- 오늘의 프롤로그
- 추천 카드
- 큐레이터의 메모
- 관심 보내기 / 넘기기
- 남은 추천 수

## 11.2 프로필 상세

상대의 이야기를 더 깊게 읽는 화면이다.

핵심 구성:

- 사진
- 기본 정보
- 인증 배지
- 나의 프롤로그
- 이야기의 목차
- 관계의 문체
- 라이프스타일 키워드
- 관심 보내기

## 11.3 매칭 리포트

왜 이 사람이 추천되었는지 설명하는 화면이다.

매칭 리포트는 프롤로그 큐레이터 캐릭터가 가장 자연스럽게 등장할 수 있는 핵심 화면이다. 단순히 점수나 조건을 나열하는 것이 아니라, 책방지기가 한 권의 책을 추천하듯 사용자가 상대의 이야기를 이해할 수 있도록 돕는다.

핵심 구성:

- 잘 맞는 이유
- 비슷한 관계 성향
- 공통 라이프스타일
- 확인해보면 좋은 차이점
- 첫 대화 추천 주제
- 프롤로그 큐레이터의 메모

문구 방향:

> 프롤로그 큐레이터가 두 사람의 관계 성향을 바탕으로 함께 읽어볼 이유를 정리했어요.

## 11.4 대화 화면

상호 관심이 발생하면 내부 대화방을 제공한다. 단, 대화방은 무기한 유지하지 않는다. 프롤로그의 대화방은 서로를 알아가기 위한 임시 공간이며, 대화를 이어갈지, 연장할지, 연락처를 교환할지 사용자가 안전하게 선택할 수 있어야 한다.

프롤로그식 카피:

> 두 사람의 첫문장이 이어졌어요.

대화방 운영 원칙:

- 기본 대화 기간은 7일로 둔다.
- 양쪽이 모두 동의하면 7일 단위로 연장할 수 있다.
- MVP 기준 최대 유지 기간은 무료 14일, Plus 21일을 가설로 둔다.
- 정식 서비스에서는 무료 21일, Plus 30일까지 검토할 수 있다.
- 연락처 교환은 상호 동의가 있을 때만 가능하다.
- 신고/차단/대화 종료는 항상 제공한다.

단, 주요 버튼은 직관적으로 유지한다.

- 대화 시작하기
- 대화 연장하기
- 연락처 교환 요청하기
- 신고하기
- 차단하기

# 13. BM 방향성 v1.1

BM은 프롤로그의 포지션과 충돌하지 않아야 한다. 프롤로그는 무한 좋아요, 부스트, 상위 노출권 중심의 데이팅 앱형 BM보다 **신뢰 기반 큐레이션 멤버십형 BM**이 적합하다.

## 13.1 기본 원칙

1. 남녀 동일 가격 또는 인증 수준 기반 가격을 우선 검토한다.
2. 남성 중심 과금 구조는 서비스의 진정성과 신뢰 이미지를 해칠 수 있으므로 신중히 접근한다.
3. 인증을 많이 한 사용자에게 더 좋은 추천 경험을 제공한다.
4. 유료화는 단순 좋아요 판매보다 추천 품질, 리포트, 고신뢰 풀, 추가 추천권과 연결한다.
5. 무료 사용자와 유료 사용자는 기본적으로 서로 매칭될 수 있다.
6. 유료 차별화는 매칭 가능 여부가 아니라 추천 수, 정보 깊이, 대화 보조, 고신뢰 풀 접근에서 만든다.

## 13.2 MVP 1차 BM 구조

MVP 1차에서는 복잡한 다단계 상품보다 다음 구조를 우선 검증한다.

| 구분 | 방향 |
|---|---|
| 무료 | 제한 추천, 요약 리포트, 상호 매칭 후 기본 대화 가능 |
| Plus | 정기 큐레이션 추천, 상세 매칭 리포트, 첫 대화 주제, 추가 추천권 혜택 |
| 추가 추천권 | 기본 추천 소진 후 추천 카드 1개를 추가 열람하는 제한형 상품 |

추가 추천권은 확실한 유료 상품으로 포함한다. 단, 무한 탐색권이 아니라 프롤로그의 큐레이션 철학에 맞는 제한형 상품이어야 한다.

## 13.3 추가 추천권 방향

추가 추천권의 초기 가격 가설은 다음과 같다.

| 상품 | 가격 가설 | 비고 |
|---|---:|---|
| 추가 추천권 1장 | 3,900원 | 추가 추천 카드 1개 열람 |
| 추가 추천권 3장 | 9,900원 | 번들 할인 |
| 추가 추천권 5장 | 추후 검토 | 사용량과 피로도 확인 후 도입 |

운영 원칙:

1. 월 구매 한도를 둔다.
2. 추천 품질이 떨어질 정도로 과도하게 판매하지 않는다.
3. 추가 추천권 구매 후에도 추천 이유와 리포트 요약을 제공한다.
4. Plus에는 월 포함권 또는 할인 혜택을 제공할 수 있다.
5. 무료 사용자도 구매 가능하되, 프롤로그 핵심 흐름을 이해한 뒤 구매하도록 설계한다.

## 13.4 장기 BM 후보

- 월 구독 Plus
- Trust Plus 또는 고신뢰 풀
- 상세 매칭 리포트
- 프로필 개선 리포트
- 추가 추천권
- 첫 대화 주제 확장
- 상호 매칭 후 타로/사주 기반 궁합 카드

## 13.5 Trust Plus 방향

Trust Plus는 MVP 이후 고신뢰 유저풀이 충분히 쌓인 뒤 도입한다. Trust Plus는 돈만 내면 들어가는 풀이어서는 안 되며, 인증 조건 충족과 결제가 함께 작동해야 한다.

가능 조건:

- 본인 인증 완료
- 얼굴 인증 완료
- 관계 목적 확인 완료
- 싱글 상태 서약 완료
- 직업/재직 인증 완료 또는 강력 권장
- 매너 위반 이력 없음

# 13.6 타로/사주 궁합 부가 콘텐츠 v1.1

상호 매칭 이후 대화를 돕는 부가 콘텐츠로 타로/사주 기반 궁합 카드를 검토할 수 있다.

# 18. 현재 기준 최종 요약

프롤로그는 **가볍지 않은 만남을 위한 인증 소개팅 앱**이다.

이 서비스는 결혼중개나 결정사 대체를 전면에 내세우지 않는다. 대신 기존 소개팅 앱의 가벼움, 허위 프로필, 목적 불명확성, 비매너 문제를 해결하기 위해 인증과 큐레이션을 결합한다.

브랜드 경험은 책방/서점 메타포를 사용한다. 사람을 조건표로만 보여주는 것이 아니라, 한 권의 이야기처럼 소개한다. 사용자는 매일 제한된 추천을 받고, 프롤로그 큐레이터의 메모를 통해 왜 이 사람이 추천되었는지 이해한다. 프롤로그 큐레이터는 실제 사람이 수동으로 추천하는 존재가 아니라, 알고리즘 기반 추천을 따뜻하고 이해하기 쉬운 언어로 설명하는 브랜드 페르소나다.

핵심 방향은 다음과 같다.

- 서비스명: **프롤로그**
- 부제: **가볍지 않은 만남을 위한 인증 소개팅**
- 대표 카피: **좋은 만남은, 좋은 첫문장에서 시작됩니다.**
- 타깃: **26\~39세, 기존 소개팅 앱의 가벼움과 불신에 피로감을 느끼는 싱글**
- 핵심 가치: **신뢰, 진정성, 큐레이션, 이야기형 프로필, 안전감**
- 핵심 경험: **인증 → 프로필 작성 → 관계 목적 확인 → 추천 카드 → 매칭 리포트 → 관심/거절 → 대화**
- 매칭 방식: **추천 기반 쌍방 관심형 매칭**
- 대화 정책: **7일 기본 대화방, 양쪽 동의 연장, 상호 동의형 연락처 교환**
- BM 방향: **무료 진입 + Plus 멤버십 + 추가 추천권 + 향후 고신뢰 풀**
- 부가 콘텐츠: **상호 매칭 후 대화 보조용 타로/사주 궁합 카드 검토**
- 장기 확장: **국내 시장 안정화 이후 한일 문화 기반 관계 매칭으로 확장**

프롤로그의 핵심 문장은 다음과 같다.

> **신뢰할 수 있는 사람을, 이야기로 만나는 소개팅.**
