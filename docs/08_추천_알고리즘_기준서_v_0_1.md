# 1. 문서 목적

이 문서는 프롤로그 MVP의 초기 추천 기준을 정의한다.

MVP 단계에서는 복잡한 머신러닝 모델보다 설명 가능한 룰 기반 추천과 운영 검증을 우선한다.

추천 알고리즘의 목적은 많은 사람을 보여주는 것이 아니라, **적은 수의 추천을 납득 가능한 이유와 함께 제공하는 것**이다.

---

# 2. 추천 철학

프롤로그는 무한 스와이프 중심의 데이팅 앱이 아니다.

추천은 사용자가 더 많은 사람을 빠르게 넘기도록 만드는 장치가 아니라, 한 사람을 더 신중하게 읽어보고 판단하도록 돕는 장치다.

따라서 프롤로그의 추천은 다음 방향을 따른다.

> 더 많은 사람을 보여주는 앱이 아니라,  
> 덜 의심해도 되는 사람을 신중하게 소개하는 앱.

---

# 3. 추천 원칙

1. 무한 탐색이 아니라 제한된 추천을 제공한다.
2. 추천 이유를 설명할 수 있어야 한다.
3. 외모와 조건만으로 추천하지 않는다.
4. 관계 목적과 만남의 온도를 중요하게 반영한다.
5. 인증 수준과 프로필 완성도를 추천 품질에 반영한다.
6. 신고/차단/제재 이력은 추천 후보 제외 또는 감점에 반영한다.
7. 사용자의 관심/거절/대화 반응을 다음 추천에 학습 데이터로 활용한다.
8. 추천은 사용자를 평가하거나 등급화하는 방식으로 노출하지 않는다.
9. 추천 품질이 낮을 때는 억지로 추천하지 않는다.
10. 유료 사용자에게도 추천 기준을 낮춘 대량 추천을 제공하지 않는다.

---

# 4. 추천 처리 단계

추천은 다음 순서로 처리한다.

1. 추천 가능 사용자 풀 생성
2. 필수 제외 조건 적용
3. 기본 필터 적용
4. 점수 계산
5. 추천 이유 생성
6. 추천 카드 노출
7. 사용자 반응 수집
8. 다음 추천에 반영

---

# 5. 추천 가능 사용자 조건

추천 후보가 되기 위한 최소 조건은 다음과 같다.

| 조건 | 필수 여부 | 설명 |
|---|---|---|
| 계정 상태 active | 필수 | 정지/탈퇴 계정 제외 |
| 본인 인증 완료 | 필수 | 가짜 계정 방지 |
| 얼굴 인증 완료 | 필수 | 도용 사진 방지 |
| 나이 확인 완료 | 필수 | 타깃 연령 확인 |
| 관계 목적 설문 완료 | 필수 | 추천 기준 필요 |
| 매너 서약 완료 | 필수 | 서비스 이용 기준 |
| 싱글 상태 서약 완료 | 필수 | 초기 신뢰 조건 |
| 대표 사진 등록 | 필수 | 프로필 기본 요건 |
| 필수 프로필 완성 | 필수 | 추천 품질 확보 |
| 신고/제재 제한 없음 | 필수 | 안전한 추천 |

---

# 6. 추천 제외 조건

다음 조건에 해당하면 추천 후보에서 제외한다.

1. 사용자가 차단한 상대
2. 상대가 사용자를 차단한 경우
3. 이미 상호 매칭된 상대
4. 최근 일정 기간 내 넘긴 상대
5. 신고 검토 중이거나 제재 상태인 사용자
6. 필수 인증 미완료 사용자
7. 프로필 검수 반려 사용자
8. 연령, 성별, 관심 대상 조건이 맞지 않는 사용자
9. 서비스 운영상 추천 제한 상태인 사용자
10. 휴면 또는 장기 미접속 사용자

# 7. 추천 점수 구조

MVP 추천 점수는 100점 만점의 룰 기반 점수로 시작한다.

| 점수 축 | 배점 | 설명 |
|---|---:|---|
| 관계 목적 적합도 | 25 | 원하는 만남, 장기 관계 가능성, 관계 속도 |
| 신뢰/인증 점수 | 20 | 인증 완료 수준, 프로필 검수 상태 |
| 라이프스타일 적합도 | 20 | 주말, 연락 빈도, 음주/흡연, 취향 |
| 대화 스타일 적합도 | 15 | 대화 빈도, 대화 깊이, 갈등 해결 방식 |
| 생활권/거리 적합도 | 10 | 지역, 실제 만남 가능성 |
| 프로필 품질 점수 | 10 | 프로필 완성도, 답변 충실도 |
| 합계 | 100 |  |

---

# 8. 추천 점수 계산 가설

MVP에서는 다음 방식으로 점수를 계산한다.

```text
총점 = 관계 목적 적합도 + 신뢰/인증 점수 + 라이프스타일 적합도 + 대화 스타일 적합도 + 생활권/거리 적합도 + 프로필 품질 점수
```

단, 점수는 사용자에게 그대로 노출하지 않는다.

사용자에게는 숫자 점수보다 다음 형태로 설명한다.

- 잘 맞는 이유
- 확인하면 좋은 차이점
- 첫 대화 추천 주제
- 큐레이터의 메모

---

# 9. 관계 목적 적합도

관계 목적 적합도는 가장 중요한 축이다.

## 9.1 반영 항목

- 원하는 만남의 형태
- 장기 관계 가능성에 대한 개방도
- 관계 시작 속도
- 연락 빈도 선호

## 9.2 점수 방향

| 조합 | 점수 방향 |
|---|---|
| 둘 다 진지한 연애/장기 관계 선호 | 높음 |
| 둘 다 자연스럽게 시작하는 연애 선호 | 중간~높음 |
| 한쪽은 가벼운 만남, 한쪽은 장기 관계 선호 | 낮음 |
| 둘 다 천천히 알아가는 관계 선호 | 높음 |
| 한쪽은 빠른 확신, 한쪽은 매우 천천히 선호 | 중간 또는 낮음 |
| 연락 빈도 기대가 유사함 | 높음 |
| 연락 빈도 기대 차이가 큼 | 낮음 또는 차이점 표시 |

## 9.3 추천 이유 문구 예시

- 두 분 모두 관계를 천천히 시작하는 편이에요.
- 두 분 모두 가볍지 않은 연애를 원한다는 점이 비슷해요.
- 관계를 시작하는 속도에는 조금 차이가 있어, 초반 대화에서 기대치를 맞춰보면 좋아요.
- 두 분 모두 장기적인 관계 가능성을 열어두고 있어요.

# 10. 신뢰/인증 점수

## 10.1 반영 항목

| 항목 | 점수 방향 |
|---|---|
| 본인 인증 | 필수 조건 |
| 얼굴 인증 | 필수 조건 |
| 관계 목적 확인 | 필수 조건 |
| 매너 서약 | 필수 조건 |
| 싱글 상태 서약 | 필수 조건 |
| 직업/재직 인증 | 가점 |
| 거주 지역 인증 | 가점 |
| 학력 인증 | 선택 가점 |
| 경제 안정성 인증 | MVP 후순위 또는 선택 가점 |

## 10.2 원칙

- 필수 인증은 점수라기보다 추천 가능 조건이다.
- 권장 인증은 추천 품질과 노출 우선순위에 반영한다.
- 인증을 많이 했다고 무조건 좋은 사람처럼 표현하지 않는다.
- 사용자에게는 “신뢰 배지” 중심으로 보여준다.
- 인증 원본 정보는 공개하지 않는다.

## 10.3 추천 이유 문구 예시

- 두 분 모두 기본 신뢰 인증을 완료한 사용자예요.
- 상대는 직업/재직 인증을 완료해 프로필 신뢰도가 높아요.
- 기본 인증과 관계 목적 확인이 완료된 프로필이에요.

## 10.4 피해야 할 문구

- 조건이 검증된 완벽한 상대예요.
- 결정사급으로 검증된 상대예요.
- 경제적으로 우수한 상대예요.
- 결혼 상대로 검증되었어요.

# 11. 라이프스타일 적합도

## 11.1 반영 항목

- 주말을 보내는 방식
- 취미/관심사
- 음주/흡연 성향
- 생활 시간대
- 외향/내향 성향
- 여행/문화생활 선호
- 반려동물 선호
- 운동/건강 관심도

## 11.2 점수 방향

| 조합 | 점수 방향 |
|---|---|
| 둘 다 조용한 주말 선호 | 높음 |
| 둘 다 활동적인 데이트 선호 | 높음 |
| 한쪽은 비흡연자 선호, 상대는 흡연 | 낮음 |
| 취미 키워드 일부 일치 | 중간 이상 |
| 생활 시간대가 크게 다름 | 낮음 또는 차이점 표시 |
| 한쪽은 술자리를 선호, 한쪽은 비음주 선호 | 낮음 또는 차이점 표시 |

## 11.3 추천 이유 문구 예시

- 주말을 보내는 방식이 비슷해요.
- 두 분 모두 조용한 대화와 산책 같은 만남에 잘 맞을 수 있어요.
- 생활 패턴은 조금 다를 수 있어 초반에 맞춰보면 좋아요.
- 취향 키워드 중 문화생활과 산책이 함께 나타나요.

# 12. 대화 스타일 적합도

## 12.1 반영 항목

- 연락 빈도
- 대화 깊이
- 유머/진지함 선호
- 갈등 해결 방식
- 애정 표현 방식
- 답장 속도 기대

## 12.2 점수 방향

| 조합 | 점수 방향 |
|---|---|
| 둘 다 꾸준한 연락 선호 | 높음 |
| 한쪽은 잦은 연락, 한쪽은 느린 연락 | 중간 또는 낮음 |
| 둘 다 차분한 대화 선호 | 높음 |
| 둘 다 유쾌한 대화 선호 | 높음 |
| 갈등 회피형과 즉시 해결형 | 중간 |
| 한쪽은 깊은 대화 선호, 한쪽은 가벼운 대화 선호 | 중간 또는 낮음 |

## 12.3 추천 이유 문구 예시

- 두 분 모두 안정적인 대화를 중요하게 보는 편이에요.
- 연락 빈도에 대한 기대는 조금 다를 수 있어요.
- 처음부터 깊은 대화를 나누기보다 천천히 분위기를 맞춰보면 좋아요.
- 둘 다 유쾌한 대화를 선호해 첫 대화의 부담이 적을 수 있어요.

# 13. 생활권/거리 적합도

## 13.1 반영 항목

- 시/도
- 시/군/구
- 주요 활동 지역
- 이동 가능 거리
- 장거리 관계 수용도

## 13.2 원칙

- MVP에서는 정밀 위치보다 행정구역 기반으로 시작한다.
- 정확한 주소나 실시간 위치는 수집하지 않는다.
- 거리는 실제 만남 가능성 판단을 위한 보조 점수로 사용한다.
- 같은 지역이라고 무조건 고득점을 주지 않고, 관계 목적/생활 패턴과 함께 판단한다.

## 13.3 점수 방향

| 조합 | 점수 방향 |
|---|---|
| 같은 시/군/구 또는 가까운 생활권 | 높음 |
| 같은 시/도이나 거리가 있음 | 중간 |
| 서로 다른 광역권 | 낮음 |
| 장거리 관계 수용도가 높음 | 거리 감점 완화 |
| 실제 만남 가능성이 낮음 | 낮음 |

## 13.4 추천 이유 문구 예시

- 생활권이 가까워 실제 만남으로 이어질 가능성이 있어요.
- 거리는 조금 있지만, 둘 다 천천히 알아가는 관계를 선호해요.
- 활동 지역이 일부 겹쳐 첫 만남 장소를 정하기 쉬울 수 있어요.

# 14. 프로필 품질 점수

## 14.1 반영 항목

- 필수 프로필 완성 여부
- 자기소개 답변 길이와 충실도
- 사진 등록 수
- 관계 성향 답변 완료 여부
- 부적절한 표현 여부
- 프로필 검수 상태

## 14.2 원칙

- 프로필이 부족한 사용자는 추천 우선순위가 낮아질 수 있다.
- 사용자를 벌주는 방식보다 프로필 개선 안내를 제공한다.
- 프로필 완성도는 매칭 리포트의 설명 품질에도 영향을 준다.
- 프로필 답변이 풍부할수록 추천 이유를 더 자연스럽게 생성할 수 있다.

## 14.3 점수 방향

| 상태 | 점수 방향 |
|---|---|
| 필수 프로필 완료 | 기본 점수 |
| 사진 2장 이상 등록 | 가점 가능 |
| 자기소개 답변 충실 | 가점 |
| 관계 문체 답변 완료 | 가점 |
| 프로필 정보 부족 | 감점 |
| 부적절한 표현 검수 중 | 추천 제한 가능 |

# 15. 추천 이유 생성 규칙

추천 이유는 점수 결과를 사용자에게 이해 가능한 문장으로 바꾸는 영역이다.

## 15.1 추천 이유 구성

1. 잘 맞는 지점 1~2개
2. 확인하면 좋은 차이점 1개
3. 첫 대화 주제 1~2개
4. 큐레이터의 메모 요약

## 15.2 문구 원칙

- 단정하지 않는다.
- 과학적으로 보장하는 듯 표현하지 않는다.
- 실제 사람이 직접 골랐다는 인상을 주지 않는다.
- 알고리즘 기반 추천이라는 사실을 투명하게 표현한다.
- 부정적 차이는 완곡하게 표현한다.
- 사용자를 점수화하거나 등급화하지 않는다.

## 15.3 좋은 문구

- 두 분은 관계를 천천히 시작하는 편이라는 점이 비슷해요.
- 주말을 보내는 방식이 비슷해 첫 만남의 부담이 적을 수 있어요.
- 연락 빈도에 대한 기대는 조금 다를 수 있어, 초반에 자연스럽게 맞춰보면 좋아요.
- 이 추천은 프로필, 관계 목적, 라이프스타일 응답을 바탕으로 생성되었어요.

## 15.4 피해야 할 문구

- 이 사람은 운명의 상대예요.
- 반드시 잘 맞을 거예요.
- 프롤로그 큐레이터가 직접 골랐어요.
- 조건상 완벽한 상대예요.
- 결혼 상대로 적합해요.
- 당신보다 수준이 높은 상대예요.
- 이 상대는 놓치면 안 돼요.

# 16. 큐레이터의 메모 생성 기준

큐레이터의 메모는 프롤로그 큐레이터가 추천 알고리즘의 결과를 사용자가 이해하기 쉬운 문장으로 풀어주는 영역이다.

## 16.1 역할

- 추천 이유를 따뜻하게 설명한다.
- 관계 성향과 라이프스타일의 연결점을 보여준다.
- 확인하면 좋은 차이점을 부담스럽지 않게 알려준다.
- 첫 대화의 힌트를 제공한다.

## 16.2 주의점

- 실제 사람이 직접 추천했다는 인상을 주지 않는다.
- 과하게 귀엽거나 유아적인 캐릭터처럼 말하지 않는다.
- 결제, 신고, 개인정보 화면에서는 큐레이터 캐릭터 사용을 줄인다.

## 16.3 예시

> 두 분은 관계를 천천히 시작하는 편이고, 주말을 보내는 방식이 비슷해요.  
> 둘 다 대화에서 안정감을 중요하게 보는 타입이라, 처음부터 과하게 빠른 관계보다는 자연스럽게 알아가는 만남에 잘 맞을 수 있어요.  
> 다만 연락 빈도에 대한 기대는 조금 다를 수 있어요.

# 17. 거절 사유 반영 규칙

사용자가 넘기기를 선택하면 거절 사유를 수집한다.

## 17.1 거절 사유별 반영 방향

| 거절 사유 | 추천 반영 방향 |
|---|---|
| 사진/분위기 호감이 낮아요 | 동일 분위기/스타일 반복 노출 완화 |
| 관계 목적이 달라 보여요 | 관계 목적 점수 가중치 조정 |
| 거리나 생활권이 맞지 않아요 | 거리/생활권 조건 강화 |
| 대화 스타일이 맞지 않을 것 같아요 | 대화 스타일 적합도 조정 |
| 직업/생활 패턴이 맞지 않아요 | 라이프스타일 점수 조정 |
| 취향이나 가치관이 달라 보여요 | 관심사/가치관 매칭 조정 |
| 프로필 정보가 부족해요 | 프로필 품질 점수 강화 |
| 기타 | 자유 의견 검토 또는 단순 스킵 처리 |

## 17.2 반영 원칙

- 한 번의 거절 사유로 과도하게 추천 방향을 바꾸지 않는다.
- 반복적으로 같은 사유가 선택될 때 가중치를 조정한다.
- 민감한 거절 사유는 상대에게 공개하지 않는다.
- 추천 개선 목적 외에 사용자를 평가하거나 낙인찍는 데 사용하지 않는다.
- 외모 관련 거절 사유는 사용자에게 노출되는 방식과 내부 활용 방식에 특히 주의한다.

# 18. 사용자 액션 반영 기준

추천 이후 사용자 액션은 다음과 같이 해석한다.

| 액션 | 의미 | 추천 반영 |
|---|---|---|
| 추천 카드 조회 | 추천 노출 | 기본 노출 로그 |
| 프로필 상세 열람 | 관심 가능성 증가 | 해당 유형 선호 신호 |
| 매칭 리포트 열람 | 추천 이유 관심 | 리포트 효용 신호 |
| 관심 보내기 | 긍정 신호 | 유사 특성 추천 강화 |
| 넘기기 | 부정 또는 비선호 신호 | 사유 기반 보정 |
| 상호 매칭 | 강한 긍정 신호 | 추천 품질 평가 |
| 첫 메시지 발송 | 대화 전환 신호 | 추천 품질 평가 |
| 대화 지속 | 높은 품질 신호 | 향후 모델 고도화 지표 |
| 신고/차단 | 강한 부정 신호 | 추천 제한/안전 점수 반영 |

# 19. 추천 수와 노출 정책

구체적인 추천 수는 BM/가격 정책과 함께 확정해야 하지만, 알고리즘 기준서는 다음 원칙을 따른다.

1. 하루 또는 주간 추천 수는 제한한다.
2. 추천 후보 풀이 부족할 때 억지 추천을 하지 않는다.
3. 추천 품질이 낮으면 “오늘은 어울리는 추천을 준비 중입니다” 빈 상태를 제공할 수 있다.
4. 추가 추천권을 사용하더라도 동일한 품질 기준을 적용한다.
5. 추가 추천권은 추천 수를 늘릴 뿐, 추천 기준을 낮추지 않는다.
6. 사용자가 추천을 모두 소진했을 때 추가 추천권 안내를 노출할 수 있다.
7. 추천권 구매를 유도하기 위해 의도적으로 기본 추천 품질을 낮추지 않는다.

---

# 20. 무료/Plus 추천 차이

무료 사용자와 Plus 사용자는 서로 매칭될 수 있다.

차이는 매칭 가능 여부가 아니라 경험의 깊이에서 만든다.

| 구분 | 무료 | Plus |
|---|---|---|
| 추천 수 | 제한 추천 | 더 많은 정기 추천 또는 우선 추천 |
| 매칭 리포트 | 요약 | 상세 |
| 첫 대화 주제 | 기본 | 확장 |
| 대화 기간 | 기본 7일, 최대 14일 가설 | 기본 7일, 최대 21일 가설 |
| 추가 추천권 | 구매 가능 | 포함권 또는 할인 가능 |
| 고신뢰 풀 | 없음 | MVP 이후 Trust Plus에서 검토 |

## 20.1 원칙

- 무료 사용자가 매칭 자체에서 배제되어서는 안 된다.
- Plus는 더 많은 기회와 더 깊은 이해를 제공하는 방향이다.
- Plus가 무한 탐색권처럼 보이면 안 된다.
- Plus 사용자에게도 추천 품질 기준은 동일하게 적용한다.

---

# 21. 추가 추천권 추천 기준

추가 추천권은 기본 추천을 모두 본 뒤, 큐레이션된 추천 카드 1개를 추가로 열람할 수 있는 제한형 유료 상품이다.

## 21.1 원칙

1. 추가 추천권은 무제한 탐색권이 아니다.
2. 추가 추천권 1장은 추가 추천 카드 1개 열람권이다.
3. 추가 추천 카드에도 추천 이유와 매칭 리포트 요약을 제공한다.
4. 추천 후보 품질이 낮으면 추가 추천권 사용을 제한할 수 있다.
5. 월 구매 한도를 둔다.
6. Plus에는 월 포함권 또는 할인 혜택을 제공할 수 있다.

## 21.2 추천 기준

- 기본 추천과 동일한 필터/점수 기준을 적용한다.
- 단, 기본 추천 소진 후에도 일정 품질 이상 후보가 있을 때만 제공한다.
- 이미 넘긴 사용자나 차단/신고 사용자는 제외한다.

---

# 22. 신고/차단/제재 반영

## 22.1 추천 제외

다음 사용자는 추천 후보에서 제외한다.

- 차단 관계가 있는 사용자
- 계정 정지 사용자
- 프로필 검수 반려 사용자
- 신고 검토 중 운영 제한이 걸린 사용자
- 반복 신고로 안전 점수가 낮은 사용자

## 22.2 감점 또는 제한

다음 경우 추천 우선순위를 낮출 수 있다.

- 반복적인 노쇼 신고
- 반복적인 무례한 대화 신고
- 프로필 정보 부족
- 인증 미완료 항목 과다
- 장기 미접속

## 22.3 주의점

- 신고가 접수되었다고 무조건 유죄처럼 처리하지 않는다.
- 신고 상태와 제재 상태를 구분한다.
- 허위 신고 가능성을 고려한다.
- 사용자에게 안전 관련 조치를 투명하게 안내하되, 신고자 보호를 우선한다.

---

# 23. 운영자 개입 범위

MVP 초기에는 추천 알고리즘이 완전 자동화되지 않을 수 있다.

운영자는 다음 범위에서 개입할 수 있다.

1. 인증 검수
2. 부적절한 프로필 제외
3. 신고 사용자 추천 제한
4. 추천 후보 부족 시 운영 보정
5. 베타 유저풀 성비/지역/연령 균형 확인
6. 추천 결과 이상 케이스 확인

## 23.1 운영자 개입 표현 원칙

운영자가 추천 후보를 보정하더라도 사용자에게 다음처럼 표현하지 않는다.

- 전담 매니저가 직접 골랐어요.
- 사람이 직접 심사해 매칭했어요.
- 상담사가 추천했어요.

권장 표현은 다음과 같다.

- 이 추천은 프로필, 관계 목적, 라이프스타일 응답을 바탕으로 생성되었어요.
- 프롤로그 큐레이터가 추천 이유를 정리했어요.

# 24. 추천 품질 평가 지표

초기 지표 설계서는 별도 문서로 다루지만, 추천 알고리즘 기준서에서는 다음 품질 지표를 참고한다.

| 지표 | 의미 |
|---|---|
| 추천 카드 열람률 | 추천이 사용자에게 읽히는지 |
| 프로필 상세 열람률 | 추천 상대에 대한 관심 정도 |
| 매칭 리포트 열람률 | 추천 이유의 효용 |
| 관심 보내기율 | 추천 긍정 반응 |
| 넘기기율 | 추천 부정 반응 |
| 거절 사유 분포 | 추천 개선 방향 |
| 상호 매칭률 | 추천 쌍방 적합성 |
| 첫 메시지 발송률 | 대화 전환 |
| 대화 지속률 | 추천 품질의 후행 지표 |
| 신고/차단률 | 안전 리스크 |

# 25. MVP 추천 로직 예시

MVP 초기에는 다음과 같은 단순 로직으로 시작할 수 있다.

```text
1. 추천 대상 사용자 중 필수 인증 완료자만 추린다.
2. 성별/관심 대상 조건을 맞춘다.
3. 나이 범위와 지역 조건을 맞춘다.
4. 차단/신고/이미 매칭/최근 스킵 사용자를 제외한다.
5. 관계 목적 적합도를 계산한다.
6. 인증/신뢰 점수를 계산한다.
7. 라이프스타일, 대화 스타일, 생활권, 프로필 품질 점수를 계산한다.
8. 총점이 높은 순으로 정렬한다.
9. 일정 기준 이상인 후보만 추천한다.
10. 추천 이유와 큐레이터 메모를 생성한다.
```

# 26. 추천 이유 생성 예시

## 26.1 관계 목적 중심

> 두 분 모두 가볍지 않은 연애를 원하고, 관계를 천천히 시작하는 편이에요.  
> 처음부터 빠르게 가까워지기보다 자연스럽게 알아가는 만남에 잘 맞을 수 있어요.

## 26.2 라이프스타일 중심

> 주말을 보내는 방식이 비슷해요.  
> 두 분 모두 조용한 카페, 산책, 전시처럼 부담스럽지 않은 시간을 선호하는 편이에요.

## 26.3 대화 스타일 중심

> 두 분 모두 안정적인 대화를 중요하게 보는 편이에요.  
> 연락 빈도에 대한 기대는 조금 다를 수 있어, 초반에 자연스럽게 맞춰보면 좋아요.

## 26.4 차이점 포함

> 관계를 시작하는 속도는 비슷하지만, 주말 활동 방식에는 조금 차이가 있어요.  
> 한 분은 조용한 휴식을, 다른 한 분은 외부 활동을 더 선호하는 편이라 첫 만남 전에 서로의 리듬을 이야기해보면 좋아요.

# 27. 향후 고도화 방향

MVP 이후 추천 알고리즘은 다음 방향으로 고도화한다.

1. 관심/거절 패턴 기반 개인화
2. 대화 시작률 기반 추천 품질 평가
3. 대화 지속률 기반 추천 품질 평가
4. 신고/차단 리스크 반영
5. 프로필 텍스트 임베딩 기반 취향/가치관 유사도
6. 지역별 유저풀 밀도 반영
7. 인증 수준별 고신뢰 풀 추천
8. Plus/Trust Plus 추천 경험 차등화
9. 추천 후보 부족 지역의 운영 보정
10. 프로필 개선 피드백과 추천 품질 연결

7. 추가 추천권 월 구매 한도

9. 신고 이력 감점 기준
