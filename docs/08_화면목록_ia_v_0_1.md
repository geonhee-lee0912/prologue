# 1. 문서 목적

이 문서는 프롤로그 MVP의 화면 목록과 정보 구조를 정의한다.

목적은 기획, 디자인, 개발이 같은 기준으로 화면 단위를 이해하고 우선순위를 정하는 것이다.

이 문서는 다음 작업의 기준 문서로 사용된다.

1. 와이어프레임 작성
2. 디자인 화면 설계
3. 기능요구사항서 상세화
4. 개발 태스크 분해
5. MVP 범위 조정
6. QA 체크리스트 작성

---

# 2. IA 설계 원칙

프롤로그의 화면 구조는 다음 원칙을 따른다.

1. 사용자가 가입 후 빠르게 인증과 프로필 작성을 완료하도록 한다.
2. 추천을 받기 전까지 필요한 조건을 명확히 안내한다.
3. 추천 경험은 무한 탐색이 아니라 제한된 큐레이션 구조로 설계한다.
4. 책방/서점 메타포는 홈, 추천 카드, 프로필, 매칭 리포트에서 활용하되 정책/안전/결제 화면에서는 명확성을 우선한다.
5. 관심 상태와 대화 가능 상태를 구분해 사용자가 매칭 진행 상황을 이해할 수 있게 한다.
6. 신고, 차단, 개인정보 공개 범위는 항상 접근 가능해야 한다.
7. MVP에서는 화면 수를 과도하게 늘리지 않고 핵심 플로우를 우선한다.

---

# 3. IA 전체 구조

프롤로그 MVP는 다음 8개 영역으로 구성한다.

1. 진입/계정
2. 인증/온보딩
3. 프로필 작성/수정
4. 홈/추천
5. 관심/매칭
6. 대화
7. 결제/추천권
8. 설정/운영/안전

---

# 4. 하단 탭 구조

MVP 하단 탭은 4개를 권장한다.

| 탭 | 역할 | 기본 화면 | 비고 |
|---|---|---|---|
| 홈 | 오늘의 추천 확인 | D01 홈 | 오늘의 프롤로그 중심 |
| 관심 | 내가 보낸/받은 관심, 매칭 상태 | E04 관심 목록 | 상호 매칭 전 상태 확인 |
| 대화 | 상호 매칭 후 대화 | G01 대화 목록 | 대화 가능 상대만 표시 |
| 마이 | 프로필, 인증, 설정 | J01 마이페이지 | 내 정보 관리 |

## 4.1 관심 탭과 대화 탭을 분리하는 이유

초기에는 관심 탭과 대화 탭을 분리하는 것을 권장한다.

이유는 다음과 같다.

1. 관심을 보낸 상태와 실제 대화 가능한 상태는 다르다.
2. 사용자가 “내가 관심을 보낸 사람”, “나에게 관심을 보낸 사람”, “서로 관심이 이어진 사람”을 구분할 수 있어야 한다.
3. 상호 매칭 전에는 대화가 열리지 않으므로, 대화 탭에 모든 관심 상태를 섞으면 혼란이 생길 수 있다.
4. 유료화나 추가 추천권이 붙을 경우 관심/추천/대화의 상태 구분이 더 중요해진다.

# 7. 영역별 화면 정의

## 7.1 진입/계정

### A01. 스플래시

목적: 앱 시작 시 브랜드를 노출하고 초기 로딩을 처리한다.

구성:

- 프롤로그 로고
- 짧은 브랜드 문구
- 로딩 상태

권장 문구:

> 좋은 만남은, 좋은 첫문장에서 시작됩니다.

---

### A02. 온보딩 소개

목적: 신규 사용자에게 프롤로그의 핵심 가치를 짧게 설명한다.

구성:

1. 인증 기반 소개팅
2. 관계 목적 확인
3. 제한된 큐레이션 추천
4. 매칭 리포트
5. 안전한 대화

CTA:

- 시작하기
- 로그인하기

---

### A03. 로그인/가입

목적: 사용자가 계정을 생성하거나 로그인한다.

신규 가입 경로 (2가지):

1. **본인 인증으로 가입** — 휴대폰 본인 인증(PASS/NICE) → 휴대폰·CI·이름·생년월일·성별 한 번에 확보
2. **카카오로 시작** — 카카오 OAuth → 추가로 본인 인증(PASS/NICE) 1회 → 동일하게 정보 확보

기존 로그인 경로 (2가지):

1. **휴대폰 OTP 로그인** — 가입 시 등록한 휴대폰 번호로 1회용 코드 검증 (PASS 재호출 없음)
2. **카카오 로그인** — 카카오 OAuth 만으로 즉시 로그인

핵심 원칙:

> 본인 인증은 가입 때 1회만. 이후 로그인은 가볍게.

---

### A04. 약관 동의

목적: 서비스 이용에 필요한 필수/선택 동의를 받는다.

구성:

- 서비스 이용약관
- 개인정보 처리방침
- 위치/알림 등 선택 동의
- 만 19세 이상 확인

본인 인증 완료 직후 진입. 동의 시점에 `UserConsent` 레코드 저장.

---

## 7.2 인증/온보딩

### B01. 인증 안내

목적: 가입 직후 사용자가 앞으로 어떤 인증 단계가 남았는지 이해하게 한다 (본인 인증은 이미 완료).

핵심 메시지:

> 본인 인증이 완료되었어요. 매칭을 받기 전에 얼굴 인증과 관계 목적 설문을 마쳐주세요.

구성:

- 완료된 인증 배지 (본인 인증)
- 남은 인증 목록 (얼굴 인증, 매너/싱글 서약, 관계 목적 설문)
- 첫 추천까지의 흐름 안내

---

### B02. 본인 인증 결과

목적: 가입 흐름에서 자동 완료된 본인 인증 결과를 보여준다 (사용자 별도 액션 없음).

구성:

- "본인 인증이 완료되었어요" 결과 배지
- 이름/생년월일은 노출하지 않음 (개인정보 보호)
- 다음 단계 (얼굴 인증) 진행 버튼

> ⚠️ 본 화면은 가입 직후 자동 표시되며, 사용자가 인증을 직접 시작하는 화면이 아니다. 본인 인증 액션은 A03 가입 흐름에서 진행된다.

---

### B03. 얼굴 인증

목적: 도용 사진과 허위 프로필 가능성을 줄인다.

구성:

- 얼굴 인증 안내
- 촬영 또는 사진 검증
- 인증 완료/실패 상태

---

### B04. 나이 확인 결과

목적: 서비스 대상 연령 조건을 확인한다 (본인 인증 결과의 생년월일 활용).

구성:

- 나이 확인 완료 안내
- 대상 연령 외 사용자 처리 안내

---

### B05. 관계 목적 설문

목적: 사용자의 만남 목적과 관계 속도를 수집한다.

질문 후보:

1. 지금 어떤 만남을 원하시나요?
2. 좋은 사람을 만난다면 어느 정도의 관계까지 생각하시나요?
3. 관계를 시작할 때 선호하는 속도는 어떤가요?
4. 대화와 연락은 어느 정도를 선호하시나요?

---

### B06. 매너 서약

목적: 노쇼, 잠수, 비매너, 허위 정보에 대한 서비스 기준을 안내하고 동의를 받는다.

구성:

- 매너 서약 문구
- 위반 시 제재 가능성 안내
- 동의 버튼

---

### B07. 싱글 상태 서약

목적: 기혼자 또는 연인 있음 리스크를 줄인다.

구성:

- 싱글 상태 서약 문구
- 허위 서약 신고/제재 안내
- 동의 버튼

---

### B08. 직업/재직 인증 안내

목적: 직업/재직 인증을 강력 권장하고 추천 품질과의 연결을 설명한다.

구성:

- 직업/재직 인증의 목적
- 공개 범위 안내
- 인증 신청 CTA
- 나중에 하기

---

## 7.3 프로필 작성/수정

### C01. 프로필 작성 시작

목적: 이야기형 프로필 작성의 의미를 안내하고 이탈을 줄인다.

권장 문구:

> 프롤로그에서는 사람을 조건표가 아니라 하나의 이야기로 소개합니다.

---

### C02. 사진 등록

구성:

- 대표 사진 업로드
- 일상 사진 추가
- 사진 가이드
- 검수 안내

---

### C03. 기본 정보 입력

구성:

- 거주 지역
- 직업군
- 키
- 흡연/음주
- 종교 등 선택 항목

주의:

- 민감할 수 있는 정보는 선택 입력으로 둔다.
- 공개 범위를 별도 설정할 수 있게 한다.

---

### C04. 나의 프롤로그 입력

구성:

- 나를 가장 잘 설명하는 문장

# 8. 주요 사용자 플로우

## Flow 1. 신규 가입 및 인증

A01 스플래시  
→ A02 온보딩 소개  
→ A03 로그인/가입  
→ A04 약관 동의  
→ B01 인증 안내  
→ B02 본인 인증  
→ B03 얼굴 인증  
→ B05 관계 목적 설문  
→ B06 매너 서약  
→ B07 싱글 상태 서약  
→ C01 프로필 작성 시작

---

## Flow 2. 프로필 작성

C01 프로필 작성 시작  
→ C02 사진 등록  
→ C03 기본 정보 입력  
→ C04 나의 프롤로그 입력  
→ C05 이야기의 목차 입력  
→ C06 관계의 문체 입력  
→ C07 프로필 미리보기  
→ D01 홈

---

## Flow 3. 추천 열람 및 관심

D01 홈  
→ D02 추천 카드  
→ D03 프로필 상세  
→ D04 매칭 리포트 요약  
→ E01 관심 보내기 완료

또는

D01 홈  
→ D02 추천 카드  
→ E02 넘기기 확인  
→ E03 거절 사유 선택

---

## Flow 4. 상호 매칭 및 대화

상대도 관심 보냄  
→ F01 상호 매칭 완료  
→ F02 안전 안내  
→ F03 첫 대화 주제  
→ G02 대화방

---

## Flow 5. 연락처 교환

G02 대화방  
→ G05 연락처 교환 요청  
→ 상대방 G06 연락처 교환 동의/거절  
→ 양쪽 동의 시 연락처 공개  
→ 대화방에 교환 완료 안내

---

## Flow 6. 신고/차단

D03 프로필 상세 또는 G02 대화방  
→ H01 신고하기 또는 H02 차단하기

→ 운영자 K04 신고 관리

# 9. 화면별 메타포 적용 기준

| 화면 영역 | 책방/서점 메타포 적용 강도 | 예시 |
|---|---|---|
| 홈 | 강함 | 오늘의 프롤로그 |
| 추천 카드 | 강함 | 큐레이터의 메모 |
| 프로필 | 강함 | 나의 프롤로그, 이야기의 목차, 관계의 문체 |
| 매칭 완료 | 중간 | 첫문장이 이어졌어요 |
| 대화 | 중간 | 첫 페이지 열기, 단 버튼은 대화 시작하기 우선 |
| 인증 | 낮음 | 명확한 인증 안내 우선 |
| 신고/차단 | 낮음 | 은유 없이 정확한 표현 |
| 개인정보 | 낮음 | 공개 범위, 보관, 삭제 명확히 표현 |
| 결제 | 낮음 | 상품/가격/혜택 명확히 표현 |
