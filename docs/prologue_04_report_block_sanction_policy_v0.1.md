# 1. 문서 목적

이 문서는 프롤로그 서비스에서 발생할 수 있는 허위 정보, 비매너 대화, 노쇼, 기혼자/연인 있음 의심, 개인정보 침해, 외부 공유, 괴롭힘 등 신뢰와 안전을 해치는 행위에 대해 신고, 차단, 검토, 제재 기준을 정의하기 위한 정책 문서다.

프롤로그는 인증 기반 진지한 소개팅 앱으로 포지셔닝한다. 따라서 사용자는 단순히 “매칭이 많이 되는 서비스”보다, “덜 의심하고 신중하게 대화를 시작할 수 있는 서비스”를 기대한다. 이 기대가 유지되려면 인증뿐 아니라 문제가 발생했을 때 작동하는 대응 체계가 필요하다.

이 문서는 다음 의사결정의 기준으로 활용한다.

1. 어떤 행위를 신고 대상으로 볼 것인지 정의한다.
2. 사용자가 언제, 어디에서 신고/차단할 수 있어야 하는지 정한다.
3. 신고 접수 후 운영 검토와 제재 단계를 정의한다.
4. 허위 신고와 보복성 신고를 방지하는 기준을 정한다.
5. 안전 UX, 개인정보 공개 범위, 대화 경험 문서와 정책 기준을 맞춘다.
6. 베타 운영 단계에서 최소한으로 필요한 운영 룰을 정한다.

이 문서는 법률 자문을 대체하지 않는다. 실제 서비스 출시 전에는 개인정보, 전자상거래, 청소년 보호, 성범죄/스토킹/명예훼손 관련 대응, 결혼중개업 오인 가능성 등에 대해 별도 법무 검토가 필요하다.

# 2. 정책 한 줄 정의

**프롤로그의 신고_차단_제재 정책은 사용자가 신뢰와 안전을 해치는 행동을 빠르게 멈추고, 서비스가 일관된 기준으로 검토하고 대응하기 위한 운영 기준이다.**

사용자에게 전달할 핵심 메시지는 다음과 같다.

> 프롤로그는 가볍지 않은 만남을 위해, 허위 정보와 비매너 행동을 가볍게 넘기지 않습니다.

---

# 3. 정책 원칙

## 3.1 사용자의 즉시 통제권을 우선한다

사용자는 불편한 상대를 발견했을 때 운영 검토를 기다리지 않고 즉시 차단할 수 있어야 한다. 차단은 사용자가 자신의 경험을 보호하기 위한 기본 권리다.

## 3.2 신고는 조용하고 안전하게 처리한다

신고 내용은 상대에게 공개되지 않아야 한다. 신고한 사용자가 보복이나 추가 연락을 걱정하지 않도록, 신고와 차단 흐름은 명확하고 조용하게 처리되어야 한다.

## 3.3 제재는 행위의 위험도와 반복성을 기준으로 한다

모든 신고가 즉시 강한 제재로 이어져서는 안 된다. 다만 허위 정보, 기혼자/연인 있음, 성희롱, 협박, 개인정보 침해, 외부 유출 등 신뢰와 안전에 큰 영향을 주는 행위는 빠르게 제한해야 한다.

## 3.4 신고자와 피신고자 모두의 권리를 고려한다

신고자는 보호받아야 하고, 피신고자도 사실과 다른 신고로 부당한 제재를 받지 않도록 최소한의 검토 기준이 있어야 한다.

## 3.5 정책은 사용자가 이해할 수 있어야 한다

신고/차단/제재 정책은 약관 속에만 숨어 있어서는 안 된다. 사용자는 어떤 행동이 금지되는지, 신고하면 어떤 일이 일어나는지, 차단하면 무엇이 달라지는지 쉽게 이해할 수 있어야 한다.

# 4. 신고와 차단의 차이

| 구분 | 목적 | 즉시 효과 | 운영 검토 | 상대방 고지 |
|---|---|---|---|---|
| 차단 | 사용자가 특정 상대와의 접점을 끊음 | 즉시 적용 | 필요 없음 | 상대에게 차단 사유는 고지하지 않음 |
| 신고 | 서비스 정책 위반 가능성을 운영팀에 알림 | 필요 시 자동 임시 제한 가능 | 필요 | 신고자 정보와 상세 내용은 상대에게 공개하지 않음 |
| 신고+차단 | 사용자를 즉시 보호하면서 운영 검토 요청 | 즉시 차단 | 필요 | 상대에게 신고자 정보 비공개 |

## 4.1 차단

차단은 사용자의 즉시 보호 기능이다. 사용자는 이유를 입력하지 않아도 차단할 수 있어야 한다.

차단 시 기본 효과:

1. 서로의 프로필 상세 접근 제한
2. 대화방 비활성화 또는 숨김 처리
3. 상대가 추가 메시지를 보낼 수 없음
4. 향후 추천에서 서로 제외
5. 상대에게 차단 사유는 표시하지 않음

## 4.2 신고

신고는 서비스 정책 위반 가능성을 운영팀에 알리는 기능이다. 신고는 제재 판단의 근거가 되므로 신고 사유, 증거, 맥락을 수집할 수 있어야 한다.

신고 시 기본 효과:

1. 신고 접수 확인
2. 신고 내용 비공개 안내
3. 필요 시 차단 동시 진행 유도
4. 위험도가 높은 신고는 자동 임시 제한 검토
5. 운영 검토 후 필요 시 제재

# 5. 신고 가능 위치

신고 기능은 사용자가 문제가 발생할 수 있는 주요 지점에서 접근 가능해야 한다.

| 위치 | 신고 가능 대상 | 우선순위 |
|---|---|---:|
| 추천 카드 | 허위 프로필, 부적절 사진, 의심 정보 | 1 |
| 프로필 상세 | 허위 정보, 기혼자 의심, 부적절 자기소개 | 1 |
| 매칭 리포트 | 추천 오류, 민감 정보 노출 의심 | 2 |
| 대화방 | 성희롱, 욕설, 괴롭힘, 외부 유도, 개인정보 요구 | 1 |
| 매칭 해제/차단 후 | 사후 신고 | 1 |
| 고객센터/마이페이지 | 일반 신고, 증빙 제출 | 2 |
| 만남 이후 피드백 | 노쇼, 비매너, 허위 정보 발견 | 2 |

# 6. 신고 사유 분류

신고 사유는 사용자가 빠르게 선택할 수 있어야 하며, 운영팀이 위험도에 따라 검토할 수 있도록 분류되어야 한다.

## 6.1 1차 신고 카테고리

| 카테고리 | 설명 | 위험도 |
|---|---|---:|
| 허위 프로필/도용 의심 | 사진, 나이, 직업, 지역 등 허위 의심 | 높음 |
| 기혼자/연인 있음 의심 | 싱글 상태 위반 의심 | 높음 |
| 부적절한 대화 | 욕설, 성희롱, 모욕, 집요한 연락 | 높음 |
| 개인정보 침해 | 연락처 강요, 캡처 공유, 외부 유출 | 높음 |
| 외부 서비스/금전 유도 | 투자, 판매, 사기, 다른 앱 유도 | 높음 |
| 노쇼/약속 파기 | 약속 후 일방적 잠수, 반복 취소 | 중간 |
| 불성실한 이용 | 매칭 후 반복 무응답, 장난성 사용 | 중간 |
| 부적절한 사진/콘텐츠 | 노출, 혐오, 폭력, 불쾌한 이미지 | 높음 |
| 기타 | 위 항목 외 불편 경험 | 상황별 |

## 6.2 세부 신고 사유 예시

### 허위 프로필/도용 의심

- 사진이 본인이 아닌 것 같아요
- 나이/지역/직업 정보가 다른 것 같아요
- 프로필 내용이 실제와 달라요
- 다른 사람의 사진이나 정보를 도용한 것 같아요

### 기혼자/연인 있음 의심

- 배우자나 연인이 있는 것 같아요
- 싱글 상태 서약과 다른 정황이 있어요
- 상대가 관계 상태를 숨긴 것 같아요

### 부적절한 대화

- 성적인 대화를 강요했어요
- 욕설이나 모욕적인 표현을 했어요
- 반복적으로 원치 않는 메시지를 보냈어요
- 불쾌하거나 위협적인 말을 했어요

### 개인정보 침해

- 연락처나 SNS 공개를 강요했어요
- 제 사진이나 대화 내용을 캡처/공유하겠다고 했어요
- 민감한 정보를 요구했어요
- 외부에서 연락을 시도했어요

### 외부 서비스/금전 유도

- 투자, 코인, 보험, 상품 구매를 권유했어요
- 유료 외부 사이트나 앱으로 이동을 유도했어요
- 금전 거래를 요구했어요
- 홍보/영업 목적으로 접근한 것 같아요

### 노쇼/약속 파기

- 약속 장소에 나타나지 않았어요
- 약속 직전 반복적으로 취소했어요
- 만남 이후 무례한 행동을 했어요

# 7. 차단 정책

## 7.1 차단 진입점

차단은 다음 위치에서 가능해야 한다.

1. 추천 카드 메뉴
2. 프로필 상세 메뉴
3. 대화방 상단 메뉴
4. 신고 완료 화면
5. 매칭 해제 화면
6. 고객센터 내 상대 프로필 신고 흐름

## 7.2 차단 전 안내 문구

차단은 사용자가 쉽게 이해할 수 있게 안내한다.

예시 문구:

> 이 사용자를 차단하면 서로의 프로필과 대화가 보이지 않으며, 앞으로 추천에도 표시되지 않습니다.

선택지:

- 차단하기
- 신고하고 차단하기
- 취소

## 7.3 차단 후 효과

차단 후에는 다음 효과가 발생한다.

1. 양방향 프로필 접근 제한
2. 기존 대화방 비활성화 또는 숨김
3. 상대의 추가 메시지 발송 차단
4. 향후 추천 제외
5. 상대에게 차단 사유 미노출
6. 사용자가 원할 경우 신고 추가 가능

## 7.4 차단 해제

차단 해제는 초기 MVP에서는 제공하지 않거나, 고객센터를 통한 제한적 해제로 둔다.

초기 권장 방향:

- 사용자의 안전을 위해 차단 해제는 즉시 셀프 처리하지 않는다.
- 실수로 차단한 경우 고객센터 문의를 통해 처리한다.
- 추후 사용자 규모와 운영 안정성에 따라 차단 목록/해제 기능을 검토한다.

# 8. 제재 단계

프롤로그의 제재는 행위의 위험도, 증거 수준, 반복성, 피해 가능성을 기준으로 단계화한다.

| 단계 | 명칭 | 내용 | 적용 예시 |
|---:|---|---|---|
| 0 | 무조치/기록 | 신고 기록만 남김 | 근거 부족, 단순 취향 불일치 |
| 1 | 주의 안내 | 정책 위반 가능성 안내 | 경미한 비매너, 불성실 대화 |
| 2 | 경고 | 공식 경고 및 재발 시 제한 안내 | 반복 무응답, 부적절 표현 경미 사례 |
| 3 | 기능 제한 | 추천/대화/프로필 노출 일부 제한 | 반복 신고, 인증 의심, 위험 행위 |
| 4 | 임시 이용 정지 | 계정 일시 정지 및 검토 | 성희롱, 허위 정보, 외부 유도 의심 |
| 5 | 영구 이용 제한 | 계정 정지 및 재가입 제한 검토 | 도용, 사기, 폭력/협박, 기혼자 악성 사례 |

# 9. 신고 유형별 기본 제재 기준

## 9.1 허위 프로필/도용 의심

| 상황 | 기본 대응 |
|---|---|
| 단순 의심, 증거 부족 | 기록, 추가 인증 요청 검토 |
| 사진 도용 정황 있음 | 얼굴 재인증 요청, 프로필 노출 제한 |
| 직업/나이/지역 허위 정황 있음 | 해당 정보 비공개 처리, 재인증 요청 |
| 명백한 도용 또는 반복 허위 | 영구 이용 제한 검토 |

## 9.2 기혼자/연인 있음 의심

| 상황 | 기본 대응 |
|---|---|
| 단순 의심 | 신고 기록, 추가 확인 가능 |
| 복수 신고 또는 정황 있음 | 계정 임시 제한, 싱글 상태 재확인 요청 |
| 증빙 자료 제출 | 운영 검토 후 강한 제재 |
| 명백한 위반 | 영구 이용 제한 검토 |

주의: 싱글 상태는 민감한 사안이므로, 사용자에게 과도한 사생활 증빙을 요구하지 않도록 별도 법무/개인정보 검토가 필요하다.

## 9.3 성희롱/욕설/괴롭힘

| 상황 | 기본 대응 |
|---|---|
| 경미한 부적절 표현 | 경고 또는 기능 제한 |
| 성적 대화 강요 | 임시 이용 정지 이상 |
| 반복적 괴롭힘 | 기능 제한 또는 영구 이용 제한 |
| 협박/스토킹/불법 촬영 암시 | 즉시 임시 제한, 필요 시 외부 기관 안내 |

## 9.4 개인정보 침해/외부 유출

| 상황 | 기본 대응 |
|---|---|
| 연락처 강요 | 경고 또는 기능 제한 |
| 캡처 공유 협박 | 임시 이용 정지 이상 |
| 실제 외부 유출 정황 | 영구 이용 제한 검토 |
| 불법 촬영/유포 위협 | 즉시 제한, 외부 신고 안내 |

## 9.5 사기/영업/외부 유도

| 상황 | 기본 대응 |
|---|---|
| 외부 앱 이동 유도 | 경고 또는 기능 제한 |
| 투자/상품 구매 권유 | 임시 이용 정지 |
| 금전 요구/사기 정황 | 영구 이용 제한 검토 |
| 조직적 홍보/스팸 | 즉시 영구 제한 검토 |

## 9.6 노쇼/비매너

| 상황 | 기본 대응 |
|---|---|
| 1회 노쇼 신고 | 기록, 주의 안내 가능 |
| 반복 노쇼 | 경고 또는 추천 제한 |
| 악의적 약속 파기 반복 | 기능 제한 |
| 만남 중 위협/성희롱 | 고위험 신고 기준 적용 |

# 10. 신고 처리 흐름

## 10.1 기본 흐름

1. 사용자가 신고 사유 선택
2. 필요 시 상세 내용 입력
3. 필요 시 대화/프로필 캡처 등 근거 자동 첨부 또는 사용자 첨부
4. 차단 여부 선택
5. 신고 접수 완료 안내
6. 운영팀 검토
7. 필요 시 추가 정보 요청
8. 제재 또는 무조치 결정
9. 신고자에게 처리 완료 또는 검토 완료 안내

## 10.2 신고 접수 완료 문구

예시:

> 신고가 접수되었습니다.  
> 신고 내용은 상대에게 공개되지 않으며, 프롤로그 운영 기준에 따라 검토됩니다.

차단을 함께 한 경우:

> 신고와 차단이 완료되었습니다.  
> 이제 해당 사용자는 회원님의 프로필과 대화에 접근할 수 없습니다.

## 10.3 처리 결과 안내 수준

신고 처리 결과는 신고자의 안전감에 중요하지만, 피신고자의 개인정보와 운영 기준도 보호해야 한다.

권장 안내 수준:

- 검토 완료 여부
- 정책 위반 여부를 일반적 수준으로 안내
- 필요한 조치가 이루어졌다는 사실
- 세부 제재 내역은 제한적으로 안내

예시:

> 신고 내용을 검토했으며, 프롤로그 운영 기준에 따라 필요한 조치를 완료했습니다.

또는

> 신고 내용을 검토했지만, 현재 제출된 정보만으로는 추가 조치가 어렵습니다. 유사한 상황이 반복되면 다시 신고해주세요.

# 11. 자동 임시 제한 기준

일부 신고는 운영 검토 전에도 사용자 보호를 위해 임시 제한이 필요할 수 있다.

자동 또는 준자동 임시 제한 후보:

1. 짧은 시간 내 다수 사용자로부터 동일 유형 신고
2. 성희롱/협박/금전 요구 키워드 포함 신고
3. 사진 도용/가짜 계정으로 복수 신고
4. 외부 링크/투자/판매 메시지 반복
5. 기혼자/연인 있음 관련 복수 신고
6. 미성년자 의심 또는 연령 허위 의심

임시 제한 내용:

- 신규 추천 중단
- 대화 발송 제한
- 프로필 노출 제한
- 재인증 요청
- 운영 검토 완료 전까지 일부 기능 정지

주의: 자동 제한은 오탐 가능성이 있으므로, MVP 단계에서는 완전 자동 제재보다 “자동 플래그 + 운영 검토” 방식을 우선한다.

# 12. 허위 신고와 악용 방지

프롤로그는 신고자의 안전을 우선하되, 허위 신고나 보복성 신고가 반복되는 상황도 방지해야 한다.

## 12.1 허위 신고 가능성이 있는 상황

- 단순 거절 이후 보복성 신고
- 대화 취향 불일치를 정책 위반으로 신고
- 근거 없이 반복적으로 여러 사용자를 신고
- 매칭 실패 또는 감정적 불만을 허위 정보로 신고

## 12.2 대응 방식

1. 신고자의 신고 이력을 참고한다.
2. 동일 사용자가 반복적으로 근거 없는 신고를 하는지 확인한다.
3. 허위 신고가 명백한 경우 신고 기능 제한 또는 주의 안내를 적용한다.
4. 단, 안전 신고는 위축되지 않도록 허위 신고 판단을 신중히 한다.

# 13. 운영자 검토 기준

운영자는 다음 기준으로 신고를 검토한다.

| 기준 | 확인 내용 |
|---|---|
| 신고 유형 | 어떤 정책 위반에 해당하는가 |
| 증거 수준 | 대화, 프로필, 첨부 자료, 반복 신고 여부 |
| 위험도 | 사용자 안전에 즉시 영향을 주는가 |
| 반복성 | 동일 사용자의 신고 이력이 있는가 |
| 고의성 | 실수인지, 반복적/악의적 행동인지 |
| 피해 가능성 | 다른 사용자에게도 위험이 확산될 수 있는가 |
| 민감성 | 개인정보, 성희롱, 사기 등 고위험 사안인가 |

# 14. 제재 고지 문구

## 14.1 주의 안내

> 프롤로그는 신뢰와 존중을 바탕으로 한 만남을 지향합니다.  
> 최근 이용 과정에서 일부 사용자가 불편을 느낄 수 있는 행동이 확인되어 안내드립니다. 이후에도 유사한 행동이 반복될 경우 서비스 이용이 제한될 수 있습니다.

## 14.2 경고

> 프롤로그 운영 기준에 위반될 수 있는 행동이 확인되었습니다.  
> 재발 시 추천, 대화, 프로필 노출 등 일부 기능 이용이 제한될 수 있습니다.

## 14.3 임시 이용 제한

> 프롤로그 운영 기준 위반 가능성이 확인되어 계정 이용이 임시로 제한되었습니다.  
> 운영팀 검토 후 결과를 안내드리겠습니다.

## 14.4 영구 이용 제한

> 프롤로그 운영 기준을 중대하게 위반한 것으로 확인되어 계정 이용이 제한되었습니다.  
> 이 조치는 서비스의 신뢰와 사용자 안전을 보호하기 위한 조치입니다.

# 15. 사용자 안내 원칙

## 15.1 신고자에게

신고자는 다음을 명확히 알아야 한다.

- 신고 내용은 상대에게 공개되지 않는다.
- 필요하면 즉시 차단할 수 있다.
- 처리 결과는 가능한 범위에서 안내된다.
- 긴급하거나 위법성이 있는 사안은 외부 기관 도움을 받을 수 있다.

## 15.2 피신고자에게

피신고자는 다음을 명확히 알아야 한다.

- 어떤 기준 위반으로 제한되었는지 일반적 수준에서 안내받는다.
- 신고자 신원은 공개되지 않는다.
- 필요한 경우 소명 또는 재인증 절차가 있을 수 있다.
- 반복 위반 시 더 강한 제재가 적용될 수 있다.
